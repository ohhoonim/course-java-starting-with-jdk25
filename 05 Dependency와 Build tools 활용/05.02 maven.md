# maven

Maven은 아파치 소프트웨어 재단에서 개발한 빌드 자동화 도구(Build Automation Tool)이자 프로젝트 관리 도구입니다. Java 프로젝트를 컴파일, 테스트, 패키징하고, 특히 <u>의존성(Dependency) 관리를 자동화</u>하는 데 핵심적인 역할을 합니다.

Maven의 핵심 철학은 <u>관례를 통한 설정(Convention over Configuration)</u>으로, 대부분의 프로젝트가 따르는 표준 디렉토리 구조와 빌드 프로세스를 정의하여 개발자가 최소한의 설정만으로도 프로젝트를 수행할 수 있게 합니다.

- POM 파일 (pom.xml): 프로젝트의 모든 정보를 담고 있는 핵심 설정 파일입니다. 이 XML 파일은 프로젝트 이름, 버전, 개발자 정보, 빌드 설정, 그리고 가장 중요한 프로젝트의 모든 의존성 목록을 정의합니다.
- 표준화된 구조: Maven은 소스 코드는 src/main/java에, 테스트 코드는 src/test/java에 있어야 한다는 등 미리 정해진 표준 디렉토리 구조를 따릅니다.

## 빌드 수명 주기 

Maven은 프로젝트의 빌드 과정을 논리적인 단계(Phase)로 나누어 관리합니다. 개발자는 단순히 단계 이름만 명령하면 됩니다.

- validate : 프로젝트가 유효한지 확인
- compile : 소스 코드를 컴파일
- test : 유닛 테스트 실행
- package : 컴파일된 코드를 JAR 또는 WAR 파일로 패키징
- install : 패키징된 파일을 로컬 저장소에 설치 (다른 로컬 프로젝트가 참조 가능)
- deploy : 최종 빌드 결과물을 원격 저장소에 배포

## 플러그인 

Maven의 모든 작업은 플러그인을 통해 수행됩니다. 컴파일, 테스트 실행, JAR 생성 모두 특정 플러그인에 의해 처리됩니다. 이를 통해 빌드 프로세스의 확장 및 사용자 정의가 가능합니다.

## Maven Wrapper

Maven Wrapper (mvnw)는 프로젝트에 특정 버전의 Maven 빌드 환경을 포함하고 관리할 수 있게 해주는 스크립트입니다. 개발자가 로컬 시스템에 Maven을 별도로 설치하거나 환경 변수를 설정하지 않아도, 해당 프로젝트의 빌드를 실행할 수 있게 해주는 도구입니다.

Maven Wrapper를 프로젝트에 적용하면 다음과 같은 파일들이 생성됩니다.

| 파일/디렉토리            | 역할                                                                                                             |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------- |
| mvnw (Linux/macOS)       | 셸 스크립트. Wrapper의 메인 실행 파일.                                                                           |
| mvnw.cmd (Windows)       | 배치 스크립트. Windows용 실행 파일.                                                                              |
| .mvn/wrapper/            | Wrapper가 사용하는 설정 및 코드가 들어있는 디렉토리.                                                             |
| maven-wrapper.properties | 사용하기로 정의된 Maven 버전 (예: distributionUrl=.../maven-3.8.6/bin/apache-maven-3.8.6-bin.zip) 정보가 기록됨. |
