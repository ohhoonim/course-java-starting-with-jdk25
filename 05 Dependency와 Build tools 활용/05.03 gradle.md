# gradle

Gradle은 Groovy와 Kotlin DSL(Domain Specific Language)을 기반으로 하는 빌드 자동화 도구(Build Automation Tool)입니다. Maven이 가진 설정의 유연성 부족과 빌드 속도 문제를 해결하기 위해 개발되었으며, 특히 대규모 및 멀티 모듈 프로젝트에서 높은 성능과 유연성을 제공합니다.

- 스크립트 기반 설정 : build.gradle 
- Task 기반 빌드 : maven처럼 고정된 빌드 수명 주기가 아닌 독립적인 Task 들의 그래프로 관리합니다. 개발자는 Task 간의 의존성만 정의하면 됩니다.
- 성능 최적화 : Gradle은 빌드 속도를 획기적으로 향상시키는 여러 기술을 기본으로 책택하고 있습니다. 

## 의존성 관리

Maven과 마찬가지로 **중앙 저장소(Maven Central)**나 다른 저장소에서 필요한 라이브러리(JAR 파일)를 자동으로 다운로드하고 관리합니다.

```groovy
// build.gradle (Groovy DSL 예시)
dependencies {
    // runtimeOnly는 실행 시에만 필요한 의존성
    runtimeOnly 'org.springframework.boot:spring-boot-starter-web' 
    // testImplementation은 테스트 시에만 필요한 의존성
    testImplementation 'org.junit.jupiter:junit-jupiter-api:5.10.2'
}
```

## 플러그인

플러그인을 적용하여 Java 컴파일, Spring Boot 실행, War 파일 생성 등 다양한 빌드 기능을 활성화합니다.

```groovy
// build.gradle (플러그인 적용 예시)
plugins {
    id 'java'
    id 'org.springframework.boot' version '3.2.0'
}
```

## 멀티 모듈 지원

대규모 프로젝트에서 여러 독립적인 모듈을 효율적으로 관리할 수 있도록 설계되었습니다. 각 모듈은 고유의 build.gradle 파일을 가질 수 있으며, 루트 설정 파일에서 모듈 간의 의존성을 쉽게 정의할 수 있습니다.

## Gradle Wrapper

Maven Wrapper와 마찬가지로, Gradle 역시 Gradle Wrapper를 사용합니다.

gradlew 스크립트가 프로젝트에 포함되어 있어, 개발자가 로컬에 Gradle을 설치하지 않아도 프로젝트가 요구하는 특정 Gradle 버전을 자동으로 다운로드하고 사용하여 빌드 일관성을 보장합니다.
