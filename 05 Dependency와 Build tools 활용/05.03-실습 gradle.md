## 망분리 환경에서 Gradle Cache를 이용한 의존성 확보 방법 

망분리된 환경(인터넷 접속이 차단된 폐쇄망)에서 Gradle이 외부 의존성(Dependency)을 Maven Central과 같은 원격 저장소에서 직접 다운로드할 수 없을 때, 미리 준비된 Gradle 캐시 폴더를 사용하여 의존성을 확보하는 방법이 가장 일반적이며 효과적입니다.

이 방법은 인터넷이 연결된 환경에서 필요한 모든 의존성을 다운로드한 후, 해당 캐시 폴더를 폐쇄망 환경으로 복사하여 재사용하는 원리입니다.


### 1\. 외부망 환경에서의 준비 (캐시 확보)

인터넷 접속이 가능한 환경(예: 개발자의 개인 PC 또는 사내 망 분리 서버)에서 빌드에 필요한 모든 의존성을 다운로드하여 캐시를 생성합니다.

#### A. 캐시 폴더 위치 확인

Gradle은 다운로드된 모든 의존성 파일(`.jar` 등)을 사용자 홈 디렉토리 내의 특정 위치에 저장합니다.

  * 경로: `$USER_HOME/.gradle/caches/modules-2/files-2.1/`

이 폴더 내에 의존성들이 GroupID/ArtifactID/Version 구조로 저장됩니다.

#### B. 의존성 다운로드 및 캐시 생성

1.  빌드 설정 파일 (`build.gradle`) 작성: 폐쇄망에서 빌드할 프로젝트의 `build.gradle` 파일을 정확하게 작성합니다. 필요한 모든 의존성(`dependencies`)과 플러그인(`plugins`)을 명시해야 합니다.

2.  클린 빌드 수행: Gradle이 캐시를 채우도록 인터넷이 연결된 상태에서 빌드를 실행합니다.

    ```bash
    gradle clean build
    ```

    이 명령을 실행하면, Gradle은 `build.gradle`에 명시된 모든 `.jar` 파일과 메타데이터 파일을 Maven Central에서 다운로드하여 로컬 캐시 폴더에 저장합니다.

### 2\. 캐시 폴더 복사 및 이동

인터넷 환경에서 생성된 캐시 폴더 전체를 압축하여 폐쇄망 환경으로 옮깁니다.

#### A. 필수 캐시 폴더 압축

빌드에 필요한 모든 의존성이 포함된 핵심 캐시 폴더를 압축합니다.

  * 주요 복사 대상: `$USER_HOME/.gradle/caches/modules-2/` 디렉토리 전체

#### B. 폐쇄망 환경으로 이동

압축된 캐시 파일(예: `gradle_caches.zip`)을 USB 드라이브나 내부망 파일 공유 시스템을 이용하여 폐쇄망 환경으로 옮깁니다.


### 3\. 폐쇄망 환경에서의 사용 (캐시 재활용)

#### A. 캐시 폴더 위치 설정

폐쇄망 환경의 빌드 서버 또는 개발자 PC에서, 복사해 온 캐시 파일의 압축을 해제하여 **동일한 Gradle 캐시 경로**에 위치시킵니다.

  * 경로: `$USER_HOME/.gradle/caches/modules-2/files-2.1/`

#### B. 빌드 실행 및 오프라인 모드 활성화

의존성 파일이 로컬 캐시에 완벽하게 준비되었다면, 이제 Gradle이 원격 저장소에 접근하지 않고 오직 로컬 캐시만 사용하도록 강제해야 합니다.

Gradle 명령어에 `--offline` 플래그를 추가하여 빌드를 실행합니다.

```bash
# --offline 플래그를 사용하여 원격 저장소 접근 시도를 차단
gradle clean build --offline
```

  * 작동 원리: `--offline` 플래그가 붙으면, Gradle은 필요한 의존성을 찾기 위해 원격 저장소에 접속하는 대신 `$USER_HOME/.gradle/caches` 폴더만 참조합니다. 이전에 외부망에서 확보해 둔 캐시 파일이 존재하므로 빌드가 성공적으로 완료됩니다.

> ⚠️ 주의사항:
>
> 1.  정확성: 외부망에서 빌드를 실행할 때 `build.gradle`에 있는 **모든** 의존성이 다운로드되도록 확실하게 확인해야 합니다. 하나라도 빠지면 폐쇄망에서 빌드가 실패합니다.
> 2.  경로 일치: 캐시 폴더를 압축 해제할 때 `$USER_HOME/.gradle/` 구조가 정확하게 유지되어야 합니다.
> 3.  버전: 사용하려는 Gradle 버전과 Java(JDK) 버전이 외부망과 폐쇄망 환경에서 일치해야 오류를 최소화할 수 있습니다.
