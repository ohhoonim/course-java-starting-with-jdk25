### 실습 4) Maven의 Dependency Managerment

- Maven의 Dependency Management에 대해 조사하시오. 

> - 버전 표준화 및 중앙 집중식 관리
>   - 목표: 프로젝트의 여러 모듈이 동일한 라이브러리(예: Spring Core, JUnit)를 사용할 때, 모든 모듈에서 같은 버전을 사용하도록 보장합니다.
>   - 작동 방식: 이 섹션에 라이브러리의 버전과 Scope를 정의하면, 각 모듈의 <dependencies> 섹션에서 해당 라이브러리를 참조할 때 버전을 생략할 수 있습니다.
>       - 만약 모듈이 버전을 생략하면, Maven은 부모 POM의 dependencyManagement에 정의된 버전을 자동으로 상속받아 사용합니다.
>       - 이를 통해 라이브러리 버전을 변경해야 할 때 단 한 곳(부모 POM)에서만 수정하면 되므로 관리가 매우 용이해집니다.

> - 멀티 모듈 프로젝트에서의 활용 (상속)
>   - 대규모 멀티 모듈 프로젝트에서 부모(Parent) pom.xml 파일에 dependencyManagement를 정의하고, 모든 하위 모듈은 이 부모 POM을 상속(inherit) 받습니다.
>   - 하위 모듈은 필요한 의존성만 <dependency> 태그에 <groupId>와 <artifactId>만 명시하고 <version>은 생략하여 부모로부터 정의된 버전을 가져옵니다.
