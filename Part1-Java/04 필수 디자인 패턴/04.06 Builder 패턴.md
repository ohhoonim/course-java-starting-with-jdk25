# 04.06 Builder 패턴

이 패턴은 복잡한 객체를 생성하는 과정과 객체의 **표현(Representation)**을 분리하여, 동일한 생성 절차를 통해 객체의 다양한 형태를 만들 수 있도록 하는 것을 목표로 합니다.

필수적인 요소 외에 선택적인 요소가 많아서 생성자(Constructor) 매개변수가 너무 많아질 때 (일명 '텔레스코핑 생성자(Telescoping Constructor)' 문제) 이 문제를 해결하기 위해 사용됩니다.

- 복잡한 객체 생성 분리: 객체의 생성 로직을 해당 객체와 분리하여 코드를 깔끔하게 만듭니다.
- 가독성 및 안전성 향상: 매개변수가 많을 때 어떤 값이 무엇을 의미하는지 명확히 알 수 있으며, 필요한 매개변수만 설정할 수 있어 코드의 가독성과 안전성이 높아집니다.
- 불변성(Immutability) 확보: 빌더 패턴으로 생성된 객체는 보통 final 필드를 가지는 불변 객체로 설계되어 스레드 안전성(Thread-Safe)에 유리합니다.


## 구현 예제

```java
// 1. Product 클래스: 최종적으로 생성될 객체
static class Computer {
    // 필수 요소
    private final String cpu;
    private final int ramGB;
    // 선택 요소
    private final String screenResolution; 
    
    // 생성자는 오직 Builder 객체만 받을 수 있도록 private으로 설정
    private Computer(Builder builder) {
        this.cpu = builder.cpu;
        this.ramGB = builder.ramGB;
        this.screenResolution = builder.screenResolution;
    }

    @Override
    public String toString() {
        return "Computer [CPU=" + cpu + ", RAM=" + ramGB + "GB, Screen=" + screenResolution + "]";
    }

    // 2. Builder 클래스 (정적 중첩 클래스): 내부적으로 Product를 생성
    public static class Builder {
        // Product와 동일한 필드를 가집니다 (Builder 패턴의 데이터 컨테이너 역할).
        // 필수 요소는 생성자에서 초기화합니다.
        private final String cpu;
        private final int ramGB;
        
        // 선택 요소는 기본값 또는 null로 초기화합니다.
        private String screenResolution = "미설정"; 

        // Builder의 생성자는 필수로 필요한 매개변수만 받습니다.
        public Builder(String cpu, int ramGB) {
            this.cpu = cpu;
            this.ramGB = ramGB;
        }

        // 3. 부품별 생성 메소드 (선택적): 자기 자신(this)을 반환하여 메서드 체이닝 가능
        public Builder setScreenResolution(String resolution) {
            this.screenResolution = resolution;
            return this;
        }

        // 4. 최종 Product 생성 메소드
        public Computer build() {
            // Builder 객체를 Product의 private 생성자에 넘겨 최종 객체 생성
            return new Computer(this); 
        }
    }
}

void main() {
    IO.println("--- Builder 패턴 활용 예제 ---");

    // 1. 모든 옵션을 설정한 복잡한 컴퓨터 생성 (가독성 우수)
    Computer gamingPC = new Computer.Builder("Intel i9", 32)
        .setScreenResolution("4K OLED")
        .build();

    IO.println("게이밍 PC: " + gamingPC);

    // 2. 선택 옵션 없이 필수 요소만 설정한 일반 컴퓨터 생성
    Computer basicPC = new Computer.Builder("AMD Ryzen 5", 16)
        // setScreenResolution 메소드를 호출하지 않아도 됨
        .build(); 

    IO.println("베이직 PC: " + basicPC);
}
```
