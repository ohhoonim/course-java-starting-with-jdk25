# junit 라이브러리

## Unit Test

유닛 테스트(Unit Test)란 소프트웨어의 가장 작은 단위(Unit)를 독립적으로 검증하는 테스트 방법입니다. 여기서 '유닛'은 일반적으로 애플리케이션의 핵심 로직을 담고 있는 클래스, 메소드 또는 함수 하나를 의미합니다.

### 유닛 테스트의 목적

  - 버그 조기 발견
  - 코드 품질 향상
  - 리펙터링 안정망
  - 문서화 역할

### 유닛 테스트 3단계(AAA패턴)

  - Arrange(준비)
  - Act(실행)
  - Assert(단언/검증)

Behaior-Driven Development, 행위 주도 개발 방법론에서 주고사용하는 Given-When-Then(GWT) 패턴 이라는 것도 있지만 핵심은 동일합니다. 

## JUnit

JUnit은 단순한 테스트 프레임워크를 넘어, 테스트 실행 환경을 관리하고 결과를 보고하는 핵심 역할, 즉 '테스트 러너(Test Runner)'로서 기능합니다.

테스트 러너 기능 제공하는 프레임으로써는 'TestNG' 라는 것도 있습니다. 


## JUnit 테스트 예제

```java
public class Calculator {
    // 덧셈 기능
    public int add(int a, int b) {
        return a + b;
    }
}
```

```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

// 테스트 클래스
public class CalculatorTest {

    @Test
    void testAdditionOfTwoPositiveNumbers() {
        /*
         * 1. Arrange (준비 / Given)
         * - 테스트에 필요한 객체와 초기 데이터 설정
         */
        Calculator calculator = new Calculator();
        int numberA = 5;
        int numberB = 3;
        int expectedResult = 8;
        

        /*
         * 2. Act (실행 / When)
         * - 테스트 대상 메소드를 실행
         */
        int actualResult = calculator.add(numberA, numberB);
        

        /*
         * 3. Assert (검증 / Then)
         * - 실행 결과가 예상 결과와 일치하는지 단언(assert)
         */
        // 두 값이 같은지 검증
        assertEquals(expectedResult, actualResult, "두 양수의 덧셈 결과가 예상과 일치해야 합니다.");
    }

    @Test
    void testAdditionWithZero() {
        // Arrange
        Calculator calculator = new Calculator();
        int numberA = 10;
        int numberB = 0;
        int expectedResult = 10;
        
        // Act
        int actualResult = calculator.add(numberA, numberB);
        
        // Assert
        assertEquals(expectedResult, actualResult, "0을 더하면 원래 값이 나와야 합니다.");
    }
}
```
