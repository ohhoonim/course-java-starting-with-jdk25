# 01.02 Java 개발을 위한 기초 지식

## void main()과 스레드 개념

이전 실습에서 아래와 유사한 코드를 작성해보았습니다. 

```java
//코드를 실행하는 void main() 
void main() {
	IO.println(“Start JDK25 coding”);
}
```

CPU 사양을 이야기할 때 ‘core’라는 단어를 볼 수 있습니다. core는 단일 처리 장치를 말합니다. 대부분의 경우 core 개수에 따라 얼마나 여러개의 일을 동시에 진행하는지 여부가 결정됩니다. 
자바의 스레드도 이와 유사합니다. 자바는 ‘main’이라는 기본 스레드로 동작하며, 코딩을 통해 main 스레드를 확장하여 스레드 개수를 늘리게 됩니다. 따라서 <u>main()은 자바에서 코드를 실행하는 기본 core</u>라고 이해하면 됩니다.  

## 변수

core는 모든 0과 1을 동시에 처리할 수 없습니다. 때에 따라 임시보관함이 필요합니다. 메모리가 이 역할을 하게 됩니다. core는 효율적인 처리를 위해 메모리를 가리키는 ‘주소’를 사용합니다. 이 주소는 직접 메모리 위치를 가리킬 수도 있고, 주소가 또다른 주소를 가리키기도 합니다.

자바에서는 ‘주소’의 역할을 하는 것이 변수입니다. 직접 주소를 가리키는 변수를 ‘원시(Primitive)’ 변수로 따로 구분을 하며, 다음과 같습니다.

- 숫자형 : byte, short, int, long
- 실수형 : float, double
- 문자형 : char
- 논리형 : boolean

주소의 주소를 가리키는 변수를 ‘참조(Reference)’ 변수라고 하며, ‘원시 변수’를 제외한 모든 변수를 통칭합니다.참조변수는 자바에서 적정한 이름을 가진 클래스(class)로 표현됩니다.
변수는 Primitive와 Reference라는 ‘형(type)’으로 구분되므로 일반적으로  ‘원시 타입’, ‘참조 타입’ 이라고 더 많이 부릅니다. 
변수는 ‘주소’만 변경하면 메모리 저장 위치를 바꿀 수 있으므로 쉽게 조작이 가능합니다. 하지만 상황에 따라 변경하면 안되는 경우가 있을 것입니다. 이럴 때 사용하되는 키워드(예약어)가 있습니다. 

- final
- static 

## 연산자

수학에서 ‘+’는 좌우의 두 값을 합치겠다라는 약속된 기호입니다. 자바에서도 약속된 기호들이 존재하는데 이를 ‘연산자(Operator)’라고 합니다. 

- 산술 연산자 : 셈을 할 때

| 연산자 | 설명               | 예시                    | 결과                     |
| ------ | ------------------ | ----------------------- | ------------------------ |
| +      | 덧셈 / 문자열 연결 | 10 + 5, "안녕" + "자바" | 15, "안녕자바"           |
| -      | 뺄셈               | 10 - 5                  | 5                        |
| *      | 곱셈               | 10 * 5                  | 50                       |
| /      | 나눗셈 (몫)        | 10 / 3                  | 3 (정수끼리 나누면 몫만) |
| %      | 나머지 (Modulus)   | 10 % 3                  | 1                        |
| ++     | 1 증가             | a++ 또는 ++a            | a의 값을 1 증가          |
| --     | 1 감소             | a-- 또는 --a            | a의 값을 1 감소          |

- 비교 연산자 : 참/거짓을 따질 때

| 연산자 | 설명        | 예시 (a=10, b=5) | 결과  |
| ------ | ----------- | ---------------- | ----- |
| ==     | 같다        | a == b           | FALSE |
| !=     | 같지 않다   | a != b           | TRUE  |
| >      | 크다        | a > b            | TRUE  |
| <      | 작다        | a < b            | FALSE |
| >=     | 크거나 같다 | a >= 10          | TRUE  |
| <=     | 작거나 같다 | b <= 5           | TRUE  |

- 논리 연산자 : 조건을 연결할 때

| 연산자 | 설명            | 예시                | 결과                       |
| ------ | --------------- | ------------------- | -------------------------- |
| &&     | AND (논리곱)    | (A > 5) && (B < 10) | 둘 다 참일 때만 true       |
| \|\|   | OR(논리합)      | A \|\| B            | 둘 중 하나만 참이어도 true |
| !      | NOT (논리 부정) | !(A > 5)            | 결과가 true면 false로 반전 |

- 대입 연산자 : 값을 저장할 때

| 연산자 | 설명          | 예시       | 축약되지 않은 형태 |
| ------ | ------------- | ---------- | ------------------ |
| =      | 단순 대입     | int a = 5; |                    |
| +=     | 더하고 대입   | a += 2;    | a = a + 2;         |
| *=     | 곱하고 대입   | a *= 3;    | a = a * 3;         |
| /=     | 나누고 대입   | a /= 2;    | a = a / 2;         |
| %=     | 나머지를 대입 | a %= 5;    | a = a % 5;         |

- 삼항 연산자 : 똑똑한 조건문
> 📋 삼항 연산자 
> 조건식 ? 참일 때 값 : 거짓일 때 값

## 배열

메모리에 데이터를 저장할 때 <u>관련있는 것들을 모아서 저장하면 데이터 탐색이 훨씬 빠를 것</u>입니다. 수학에서는 관련 있는 것들을 모아둔 것을  ‘집합’이라고 합니다만, 프로그래밍에서는 자료 구조라고 합니다. (미묘한 차이가 있습니다)

자료구조의 가장 기본이 되는 형태가 배열입니다. <u>중괄호(“{}”)를 이용하여 배열을 작성하고 변수 선언시 대괄호(“[]”)를 붙여주어 배열을 담고 있음을 알려줍니다.</u> 자바에서는 다른 프로그래밍 언어와는 다르게 동일한 유형의 자료들로만 배열로 만들 수 있습니다. 

```java
// 숫자만 모아놓은 배열
void main() {
	int[] numbers = {1, 2, 3, 4};
}
```

배열에는 ‘인덱스(index)’라고 하는 고유번호가 순서대로 지정이 됩니다. 시작은 ‘0’부터 시작하며, 인덱스를 이용하여 특정 위치에 있는 데이터 값을 꺼내 올 수 있습니다. 

```java
// 배열에서의 인덱스
void main() {
	int[] numbers = {1, 2, 3, 4, 5};
	IO.print(numbers[0];    // 숫자 1을 출력
}
```
