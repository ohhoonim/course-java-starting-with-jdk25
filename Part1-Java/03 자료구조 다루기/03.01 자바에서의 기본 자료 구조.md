# 자바에서의 기본 자료 구조

자바에서는 자료구조를 Collection 프레임워크라는 이름으로 표준화된 구조를 제공합니다. 

## Stack 

스택은 LIFO(Last-In, First-Out) 원칙을 따릅니다. 즉, 가장 나중에 들어간 데이터가 가장 먼저 나옵니다. 접시를 쌓아 올리는 것에 비유할 수 있습니다. 가장 위에 있는 접시(가장 최근에 넣은 데이터)를 먼저 꺼내야만 아래 접시에 접근할 수 있습니다.

Stack에서 필요한 행동은 다음과 같습니다. 

- Push (삽입): 데이터를 스택의 가장 위에 추가합니다.
- Pop (삭제): 스택의 가장 위에 있는 데이터를 제거하고 반환합니다.
- Peek (확인): 스택의 가장 위에 있는 데이터를 제거하지 않고 확인합니다.

```java
void main() {
    Stack<String> cardStack = new Stack<>();

    // 1. Push: 데이터 추가 (쌓기)
    IO.println("--- 데이터 Push (쌓기) ---");
    cardStack.push("첫 번째 카드");
    cardStack.push("두 번째 카드");
    cardStack.push("세 번째 카드 (가장 나중에 들어옴)");
    IO.println("현재 스택: " + cardStack);
    // 출력: [첫 번째 카드, 두 번째 카드, 세 번째 카드 (가장 나중에 들어옴)]

    // 2. Peek: 가장 위 데이터 확인 (제거하지 않음)
    IO.println("\n--- 데이터 Peek (확인) ---");
    IO.println("가장 위 카드 확인: " + cardStack.peek());
    IO.println("Peek 후 스택 크기: " + cardStack.size()); // 크기는 그대로 3

    // 3. Pop: 데이터 제거 및 반환 (가장 나중에 들어온 것부터 나감)
    IO.println("\n--- 데이터 Pop (꺼내기) ---");
    IO.println("꺼낸 카드 (1): " + cardStack.pop()); // LIFO: 세 번째 카드
    IO.println("꺼낸 카드 (2): " + cardStack.pop()); // 두 번째 카드
    IO.println("현재 스택: " + cardStack);
}

```

## Queue

큐는 FIFO(First-In, First-Out) 원칙을 따릅니다. 즉, 가장 먼저 들어간 데이터가 가장 먼저 나옵니다. 줄 서기에 비유할 수 있습니다. 가장 먼저 줄을 선 사람(가장 먼저 들어온 데이터)이 먼저 처리됩니다

Queue에서 필요한 행동은 다음과 같습니다. 

- Enqueue (삽입): 데이터를 큐의 뒷부분(Rear)에 추가합니다. (Java에서는 add 또는 offer)
- Dequeue (삭제): 큐의 앞부분(Front)에 있는 데이터를 제거하고 반환합니다. (Java에서는 remove 또는 poll)
- Peek (확인): 큐의 앞부분에 있는 데이터를 제거하지 않고 확인합니다. (Java에서는 element 또는 peek)

```java

void main() {

    // Queue는 인터페이스이므로 LinkedList 등의 구현체를 사용합니다.
    Queue<String> waitingQueue = new LinkedList<>();

    // 1. Enqueue (offer): 데이터 추가 (줄 서기)
    IO.println("--- 데이터 Enqueue (줄 서기) ---");
    waitingQueue.offer("첫 번째 손님 (가장 먼저 들어옴)");
    waitingQueue.offer("두 번째 손님");
    waitingQueue.offer("세 번째 손님");
    IO.println("현재 대기열: " + waitingQueue);
    // 출력: [첫 번째 손님 (가장 먼저 들어옴), 두 번째 손님, 세 번째 손님]

    // 2. Peek: 가장 앞 데이터 확인
    IO.println("\n--- 데이터 Peek (확인) ---");
    IO.println("가장 앞 손님 확인: " + waitingQueue.peek());
    IO.println("Peek 후 대기열 크기: " + waitingQueue.size()); // 크기는 그대로 3

    // 3. Dequeue (poll): 데이터 제거 및 반환 (가장 먼저 들어온 것부터 나감)
    IO.println("\n--- 데이터 Dequeue (처리) ---");
    IO.println("처리된 손님 (1): " + waitingQueue.poll()); // FIFO: 첫 번째 손님
    IO.println("처리된 손님 (2): " + waitingQueue.poll()); // 두 번째 손님
    IO.println("현재 대기열: " + waitingQueue);
}

```


## List (순서가 있고 중복허용)

데이터의 순서가 중요하며, 동일한 데이터를 여러 번 저장할 수 있습니다. 배열과 유사하지만 크기가 동적으로 변합니다

- ArrayList
- LinkedList

```java
void main() {
    // List 인터페이스를 ArrayList 구현체로 생성 (순서 O, 중복 O)
    List<String> orderList = new ArrayList<>();

    // 1. 데이터 추가 (중복 허용)
    IO.println("--- List: ArrayList ---");
    orderList.add("커피");
    orderList.add("샌드위치");
    orderList.add("커피");
    IO.println("초기 주문 목록: " + orderList);

    // 2. 인덱스를 이용한 데이터 접근
    String secondItem = orderList.get(1);
    IO.println("두 번째 주문 항목: " + secondItem);

    // 3. 데이터 삭제
    orderList.remove(0);
    IO.println("삭제 후 목록: " + orderList);
}

```

## Set (중복허용 안됨)

데이터의 중복을 허용하지 않습니다. 데이터가 있는지 없는지 빠르게 확인(검색)하는 데 주로 사용됩니다.

- HashSet
- TreeSet

```java
void main() {
    // Set 인터페이스를 HashSet 구현체로 생성 (순서 X, 중복 X)
    Set<String> uniqueCustomers = new HashSet<>();

    // 1. 데이터 추가
    IO.println("\n--- Set: HashSet ---");
    uniqueCustomers.add("김철수");
    uniqueCustomers.add("이영희");
    boolean isAdded = uniqueCustomers.add("김철수"); // 중복 데이터 추가 시도 (무시됨)

    IO.println("초기 고객 목록: " + uniqueCustomers);
    IO.println("김철수 재추가 성공 여부: " + isAdded); // false

    // 2. 데이터 존재 여부 확인
    boolean hasLee = uniqueCustomers.contains("이영희");
    IO.println("이영희 고객이 목록에 있는가? " + hasLee);

    // 3. 데이터 삭제
    uniqueCustomers.remove("이영희");
    IO.println("삭제 후 목록: " + uniqueCustomers);
}

```

## Map (키-값 쌍)

데이터를 **키(Key)**와 **값(Value)**의 쌍으로 저장하며, 키는 중복될 수 없습니다.

- HashMap
- TreeMap

```java
void main() {
    // Map 인터페이스를 HashMap 구현체로 생성 (키-값 쌍, 키 중복 X)
    Map<String, Integer> studentScores = new HashMap<>();

    // 1. 데이터 추가 (put)
    IO.println("\n--- Map: HashMap ---");
    studentScores.put("홍길동", 90);
    studentScores.put("강감찬", 85);
    studentScores.put("홍길동", 95); // 키 중복 시 값 덮어씀

    IO.println("현재 학생 점수: " + studentScores);

    // 2. 키를 이용한 값 조회 (get)
    Integer hongScore = studentScores.get("홍길동");
    IO.println("홍길동의 최종 점수: " + hongScore);

    // 3. 키 목록 순회
    IO.println("--- 모든 학생 출력 ---");
    for (String name : studentScores.keySet()) {
        Integer score = studentScores.get(name);
        IO.println("학생 이름: " + name + ", 점수: " + score);
    }

    // 4. 데이터 삭제
    studentScores.remove("강감찬");
    IO.println("강감찬 삭제 후 Map: " + studentScores);
}

```

## Iterable과 Iterator

Collection framework의 모든 자료구조는 Iterable 인터페이스를 상속하여 컬렉션의 요소들을 순서대로 접근하는 방법을 정의(코딩)하도록 강제합니다. 이 정의는 Iterator 객체를 반환하도록 하여 다음의 메소드를 사용하여 컬렉션 요소들을 탐색하고 제거하는 표준화된 방법을 제공합니다.

Iterable하다라고 하는 것은 반복문을 사용할 수 있다는 의미를 내포하고 있습니다.

- hasNext(): 순회할 다음 요소가 있는지 **boolean**으로 반환합니다.
- next(): 다음 요소를 반환하고 커서(포인터)를 다음 위치로 이동시킵니다.
- remove(): next()로 반환된 마지막 요소를 컬렉션에서 제거합니다. (순회 중 요소 제거 시 안전한 방법)

```java
void main() {
    // 1. HashSet 생성 (java.util 패키지 자동 import)
    Set<String> memberNames = new HashSet<>();

    IO.println("--- Set 활용 및 Iterator 사용 ---");

    // 2. 데이터 추가 (중복 허용 안됨)
    memberNames.add("Alice");
    memberNames.add("Bob");
    memberNames.add("Charlie");
    memberNames.add("Alice"); // 중복되므로 추가 무시됨

    IO.println("초기 멤버 수 (중복 제거됨): " + memberNames.size()); // 3
    IO.println("멤버 목록: " + memberNames);

    // 3. Iterator를 이용한 안전한 순회 및 제거
    // 목표: 이름 길이가 3을 초과하는 멤버만 제거 (즉, Alice와 Bob 제거)
    Iterator<String> iterator = memberNames.iterator();

    while (iterator.hasNext()) {
        String name = iterator.next();

        if (name.length() > 3) {
            // Iterator의 remove() 사용: 순회 중 요소를 안전하게 제거하는 유일한 방법
            iterator.remove();
            IO.println(" > 제거된 멤버 (길이 > 3): " + name);
        }
    }

    // 4. 최종 결과 확인
    IO.println("\n순회 및 제거 후 최종 멤버 목록: " + memberNames);
}

```

## Comparator와 Comparable

Comparator와 Comparable은 Java에서 객체 간의 정렬 순서를 정의하기 위해 반드시 필요한 두 가지 인터페이스입니다. 이들은 컬렉션에 저장된 사용자 정의 객체(User-Defined Objects)를 우리가 원하는 기준으로 비교하고 정렬할 수 있게 해줍니다

Java의 기본 자료형(예: int, String, double)은 크기 비교나 알파벳 순서와 같은 자연적인 정렬 순서(Natural Ordering)가 이미 정의되어 있습니다. 하지만, 우리가 만든 Student나 Product와 같은 클래스는 JVM이 정렬 기준을 알 수 없습니다.

```java
// 정렬 대상 객체: Comparable을 구현하여 자연적인 순서(점수 내림차순) 정의
class Student implements Comparable<Student> {
    String name;
    int score;

    Student(String name, int score) {
        this.name = name;
        this.score = score;
    }

    @Override
    public String toString() {
        return name + " (" + score + "점)";
    }

    // Comparable 구현: 기본 정렬 기준 (점수 내림차순)
    // 현재 객체(this)의 점수가 매개변수 객체(other)보다 작으면 양수 반환 (내림차순)
    @Override
    public int compareTo(Student other) {
        return other.score - this.score;
    }
}

void main() {
    // 1. List 생성 및 Student 객체 추가
    List<Student> students = new ArrayList<>();
    students.add(new Student("김민수", 80));
    students.add(new Student("박서준", 95));
    students.add(new Student("이지은", 80));
    students.add(new Student("최유나", 70));

    IO.println("--- 초기 목록 (추가된 순서) ---");
    IO.println(students);

    // 2. Comparable을 이용한 정렬 (자연적인 순서: 점수 내림차순)
    Collections.sort(students);

    IO.println("\n--- 1차 정렬 (Comparable: 점수 내림차순) ---");
    IO.println(students);
    // 출력 결과는 점수가 높은 순으로 정렬됩니다. (95점, 80점, 80점, 70점)

    // 3. Comparator를 이용한 정렬 (추가적인 순서: 이름 오름차순)
    // 정렬 기준을 람다식으로 즉시 정의 (클래스 외부 정렬 기준)
    students.sort((s1, s2) -> s1.name.compareTo(s2.name));

    IO.println("\n--- 2차 정렬 (Comparator: 이름 오름차순) ---");
    IO.println(students);
    // 출력 결과는 이름의 사전 순으로 정렬됩니다. (김민수, 박서준, 이지은, 최유나)
}

```
