## 실습

### 실습1) Map은 Iterable하지 않으므로 직접적으로 반복문을 사용할 수 없다.  

- entrySet()을 이용하여 반복문을 사용하는 코드를 작성하시오.

```java
void main() {
    Map<String, Integer> scores = new HashMap<>();
    scores.put("Alice", 90);
    scores.put("Bob", 85);

    // Map 자체는 순회할 수 없지만, entrySet() 뷰는 순회 가능합니다.
    for (Map.Entry<String, Integer> entry : scores.entrySet()) {
        String key = entry.getKey();
        Integer value = entry.getValue();
        IO.println("학생: " + key + ", 점수: " + value);
    }
}
```

### 실습2) Comparator와 Comparable의 차이

- Comarator와 Comparable은 둘 다 정렬을 정의하도록 합니다
- 이 둘의 차이점을 조사하시오. 

> 📋 Comparator와 Compaable의 차이 
> - Comparable (자연적인 정렬 순서 정의)
>     - 목적: 클래스 자신이 "나를 어떻게 정렬해야 하는가"라는 단 하나의 기본 정렬 기준을 정의하도록 합니다.
>     - 활용: 컬렉션(TreeSet, TreeMap)에 객체를 저장하거나, Collections.sort() 또는 Arrays.sort()를 호출할 때 객체 내부의 정렬 기준이 사용됩니다.
>     - 구현: 객체 내부에 compareTo(T other) 메소드를 구현합니다.
> - Comparator (외부적인 정렬 기준 정의)
>     - 목적: 클래스 외부에서 "이 객체를 다른 다양한 기준으로 어떻게 정렬해야 하는가"를 정의하도록 합니다. 객체에는 손대지 않고 여러 개의 정렬 기준(예: 이름 기준, 나이 기준, 점수 기준)을 만들 수 있습니다.
>     - 활용: Collections.sort()나 List.sort()를 사용할 때 정렬 기준을 인수로 넘겨줄 때 사용됩니다.
>     - 구현: 별도의 클래스나 람다식으로 compare(T o1, T o2) 메소드를 구현합니다.
