# 03.03 Stream

## 표준 FunctionalInterface 4종

아래 표기된 4종 외에 확장된 FunctionalInterface은 `java.util.function` 패키지에 정의되어있습니다. 

> 📋 java.util.function
> https://docs.oracle.com/en/java/javase/25/docs/api//java.base/java/util/function/package-summary.html

FunctionalInterface 는 메소드를 하나만 정의할 수 있으므로 api 문서를 읽을 때 메서드 명은 중요하지 않습니다.

| 인터페이스 이름  | 추상 메소드       | 매개변수 | 반환 타입   | 용도               |
| ---------------- | ----------------- | -------- | ----------- | ------------------ |
| `Supplier<T>`    | T get()           | X        | O (T)       | 값을 제공 (생산)   |
| `Consumer<T>`    | void accept(T t)  | O (T)    | X (void)    | 값을 소비 (수용)   |
| `Predicate<T>`   | boolean test(T t) | O (T)    | O (boolean) | 조건을 검사 (판단) |
| `Function<T, R>` | R apply(T t)      | O (T)    | O (R)       | 값을 변환 (적용)   |

## Supplier

파라미터가 없고, 특정타입의 값을 return 하는 타입입니다.

```java
void main() {
    // Supplier<String>: String을 공급
    Supplier<String> greeting = () -> "안녕하세요, Supplier입니다.";
    
    IO.println(greeting.get()); 
}
```

## Consumer

파라미터가 있고, 어떠한 값도 반환(return)하지 않는 타입입니다. 

```java
void main() {
    // Consumer<Integer>: Integer를 소비
    Consumer<Integer> printAndLog = number -> {
        IO.println("소비된 값: " + number);
        // 여기에 로깅 등의 추가 작업을 수행할 수 있습니다.
    };
    
    printAndLog.accept(100); 
}
```


## Predicate

파라미터가 있고, 참/거짓의 boolean을 반환하는 타입입니다. 

```java
void main() {
    // Predicate<String>: String을 테스트하여 길이가 5를 초과하는지 검사
    Predicate<String> isLongName = name -> name.length() > 5;
    
    IO.println("Alice (길이 5 초과?): " + isLongName.test("Alice"));   // false
    IO.println("Jonathan (길이 5 초과?): " + isLongName.test("Jonathan")); // true
}
```

## Function

파라미터가 있고, 반환할 타입을 지정하여 반환합니다.

```java
void main() {
    // Function<String, Integer>: String을 Integer로 변환 (문자열 길이 반환)
    Function<String, Integer> stringToLength = str -> str.length();
    
    Integer length = stringToLength.apply("Functional");
    IO.println("문자열 'Functional'의 길이: " + length); // 12
}
```

## Method Reference

람다식은 FunctionalInterface를 간결하게 표현하지만 단순히 어떤 객체의 기존 메소드를 호출하는 형태라면 Method Reference로 더 간결하게 표현할 수 있다

| 유형                        | 문법                     | 설명                                                      | 예시                                                       |
| --------------------------- | ------------------------ | --------------------------------------------------------- | ---------------------------------------------------------- |
| 정적 메소드                 | 클래스명::정적메소드     | 클래스의 정적 메소드를 참조합니다.                        | Integer::parseInt <br>(람다: s -> Integer.parseInt(s))     |
| 특정 객체의 인스턴스 메소드 | 객체참조::인스턴스메소드 | 이미 생성된 특정 객체 인스턴스의 메소드를 참조합니다.     | System.out::println <br>(람다: s -> System.out.println(s)) |
| 특정 타입의 인스턴스 메소드 | 클래스명::인스턴스메소드 | 첫 번째 인자가 메소드를 실행하는 객체(인스턴스)가 됩니다. | String::length <br>(람다: s -> s.length())                 |
| 생성자                      | 클래스명::new            | 클래스의 생성자를 참조합니다.                             | ArrayList::new <br>(람다: () -> new ArrayList())           |

```java
void main() {
    List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
    names.forEach(System.out::println);
    // names.forEach(s -> System.out.println(s)); 와 동일
}
```


## Stream

스트림은 배열, 컬렉션(List, Set)의 데이터를 처리하기위해 함수형 연산을 지원하는 클래스입니다. 

> 📋 java.util.stream package 
> https://docs.oracle.com/en/java/javase/25/docs/api//java.base/java/util/stream/package-summary.html

스트림을 이용한 데이터 처리는 일반적으로 다음 세 가지 단계로 구성됩니다.

- 데이터 소스 : Arrays.stream(array) 
- 중간 연산 : filter(), map(), sorted(), distinct()
- 중단 연산 : forEach(), collect(), sum(), count(), redouce()

종단 연산이 실행된 후에는 stream 을 재사용할 수 없습니다.

```java
void main() {
    // 1. 데이터 소스 생성: List.of()를 사용하여 불변 리스트를 즉시 생성
    List<String> fruits = List.of("apple", "banana", "grape", "avocado", "orange");
    IO.println("원본 리스트: " + fruits);

    // 2. Stream 파이프라인 구성
    List<String> processedList = fruits.stream()
            // [중간 연산 1: filter] 'a'로 시작하는 요소만 필터링
            .filter(fruit -> fruit.startsWith("a"))

            // [중간 연산 2: map] 모든 요소를 대문자로 변환
            .map(String::toUpperCase) // 메소드 참조 사용

            // [중간 연산 3: sorted] 알파벳 순으로 정렬
            .sorted()

            // [종단 연산: collect] 처리된 스트림 요소를 다시 List로 수집
            .collect(Collectors.toList());

    IO.println("\n--- 최종 결과 ---");
    IO.println("처리된 리스트 (a로 시작, 대문자, 정렬): " + processedList);
    // 예상 출력: [APPLE, AVOCADO]
}

```

## Collector와 Collectors

Collector는 스트림의 종단 연산에서 사용되며, 스트림의 요소들을 원하는 형태로 결과를 만들어 주는 방법을 정의하는 인터페이스입니다. 종단 연산을 Collector를 이용하여 구현하는 것도 좋지만 Collecotrs라는 미리 구현된 클래스를 이용하면 편리합니다. 아래 메서드외에도 많은 기능을 제공해주고 있습니다. 

- Collectors.toList()
- Collectors.toSet()
- Collectors.toMap()
- Collectors.joining()
- Collectors.counting()

## Gatherer와 Gatherers

Gatherer는 스트림의 중간 연산을 정의하는 인터페이스입니다. 이미 제공되고 있는 중간 연산(map, filter)만으로는 구현하기 어려웠던 복잡한 상태 기반의 스트림 중간 연산을 가능하게 합니다.

Collector와 마찬가지로 Gatherers 라는 유틸 클래스를 사용하면 편리합니다.

- Gatherers.mapConcurrent(int, Function)
- Gatherers.scan(identity, accumulator)
- Gatherers.windowFixed(int)
- Gatherers.windowSliding(int)
- Gatherers.fold(identity, accumulator)

```java
void main() {
    // will contain: ["1", "12", "123", "1234", "12345", "123456", "1234567", "12345678", "123456789"]
    List<String> numberStrings = Stream.of(1, 2, 3, 4, 5, 6, 7, 8, 9)
            .gather(Gatherers.scan(() -> "", (string, number) -> string + number))
            .toList();
    IO.println(numberStrings);
}

```
