# Generic

## Object class 와 상속 

> 📋 java.lang.Object
> https://docs.oracle.com/en/java/javase/25/docs/api//java.base/java/lang/Object.html

Java에서 어떤 클래스를 선언하든, 명시적으로 다른 클래스를 상속하지 않는다면 그 클래스는 자동으로 java.lang.Object 클래스를 상속받습니다.이러한 특성 때문에 Object는 Java의 모든 클래스(인터페이스 제외)의 직간접적인 부모가 됩니다.

Object 클래스는 Java에서 존재하는 모든 객체가 가져야 하는 가장 기본적인 행동들을 정의하고 구현합니다. 즉, 어떤 객체를 만들든, 그 객체는 Object가 가진 핵심 메소드들을 사용할 수 있습니다.

- toString()
    - 목적 :객체의 정보를 문자열로 표현합니다. 
    - 상속시 활용 :객체를 출력하거나 로그로 남길 때, 객체의 상태(필드 값)를 쉽게 확인할 수 있도록 재정의합니다. 
- equals(Object obj)
    - 목적 : 현재 객체와 매개변수로 넘어온 객체가 동일한지 여부를 비교합니다. 
    - 상속시 활용 : 두 객체의 논리적인 동일성 (예: ID나 이름이 같은지)을 비교하도록 재정의합니다. (==은 참조 동일성 비교) 
- hasCode()
    - 목적 :  객체의 해시 코드 값(정수)을 반환합니다.
    - 상속시 활용 : equals()가 true를 반환하는 두 객체는 반드시 같은 hashCode()를 반환하도록 equals()와 함께 재정의해야 합니다. (HashMap, HashSet 사용 시 필수) 
- getClass()
    - 목적 : 객체의 실제 런타임 클래스(Class 객체)를 반환합니다. 
    - 상속시 활용 : 객체의 타입을 실행 시점에 확인(Reflection)하는 데 사용됩니다. 
- clone()
    - 목적 : 객체의 복사본을 생성합니다. 
    - 상속시 활용 : 객체의 상태를 복사하여 새로운 인스턴스를 만들 때 사용합니다. (Cloneable 인터페이스 필요) 
- notify(), waite()
    - 목적 :  멀티스레드 환경에서 스레드 간 동기화 및 통신을 위해 사용됩니다.
    - 상속시 활용 : 특정 조건이 만족될 때까지 스레드를 대기시키거나(wait), 대기 중인 스레드를 깨울 때(notify) 사용됩니다. 

## 클래스 상속과 인터페이스 상속

클래스 상속은 'extends'키워드를 사용하며 자식 클래스가 부모 클래스의 구현을 물려받아 확장합니다. 'abstract' 키워드를 사용하여 자식이 구현하게 할 수 도 있습니다.

인터페이스 상속은 'implements' 키워드를 사용하며, 자식 클래스는 부모 인터페이스가 정의(시그니처)한 메소드를 구현해야합니다,

클래스 상속은 단일 부모만 가능하며, 인터페이스는 여러 부모를 가질 수 있으므로 객체의 다형성을 표현하는데 유리합니다

인터페이스 상속을 통한 객체 타입의 확장은 '필수 디자인 패턴'에서 깊게 알아볼 예정입니다. 

## Generic과 Collection

```java

interface Movable {
    void move();
}


class Car implements Movable {
    void startEngine() {
        IO.println("자동차 시동을 겁니다");
    }

    @Override
    public void move() {
        IO.println("자동차로 이동합니다");
    }
}


class Bicycle implements Movable {
    @Override
    public void move() {
        IO.println("자전거로 이동합니다.");
    }

}
void main() {
    // 1. 다형성(Polymorphism) 활용
    // List<Movable> : Movable 인터페이스를 구현한 모든 객체를 하나의 리스트에 담을 수 있습니다.
    List<Movable> movables = new ArrayList<>();
    
    Car myCar = new Car();
    Bicycle myBicycle = new Bicycle();
    
    movables.add(myCar); 
    movables.add(myBicycle); 
    
    IO.println("--- Movable 객체 목록 실행 ---");

    // 2. 일괄 처리
    // 리스트를 순회하며 move() 메소드를 호출합니다.
    // 각 객체는 자신에게 맞는 move() 메소드 구현을 실행합니다.
    for (Movable item : movables) {
        // item이 Car인지 Bicycle인지 몰라도, move() 기능은 반드시 가지고 있음을 보장합니다.
        item.move(); 
    }
    
    IO.println("\n--- 인터페이스의 한계 확인 ---");
    // 3. 인터페이스 참조의 한계
    // Movable 타입의 변수 item으로는 Car나 Bicycle의 고유 메소드(예: startEngine)를 호출할 수 없습니다.
    // item.startEngine(); // 🚨 컴파일 에러 발생! Movable에는 startEngine()이 정의되어 있지 않음.
    
    // 하지만 원래 타입의 변수(myCar)로는 고유 메소드 호출이 가능합니다.
    myCar.startEngine();
}
```

## Generic 클래스

```java
// T (Type)라는 타입 매개변수를 받는 Generic 클래스
class Box<T> {
    private T content; // content 필드의 타입이 T로 일반화됨

    public void setContent(T content) {
        this.content = content;
    }

    public T getContent() {
        return this.content;
    }
}

void main() {
    IO.println("--- Generic 클래스 Box<T> 활용 ---");

    // 1. String 타입으로 Generic Box 사용
    // T가 String으로 결정되어, Box에는 String만 담을 수 있습니다.
    Box<String> stringBox = new Box<>();
    stringBox.setContent("제네릭 문자열");
    // stringBox.setContent(123); // 🚨 컴파일 에러 발생!

    String strContent = stringBox.getContent();
    IO.println("String Box 내용: " + strContent);
    IO.println("꺼낸 데이터 타입: " + strContent.getClass().getName());


    // 2. Integer 타입으로 Generic Box 사용
    // T가 Integer로 결정되어, Box에는 Integer만 담을 수 있습니다.
    Box<Integer> integerBox = new Box<>();
    integerBox.setContent(42);

    Integer intContent = integerBox.getContent();
    IO.println("\nInteger Box 내용: " + intContent);
    IO.println("꺼낸 데이터 타입: " + intContent.getClass().getName());

    // 3. 타입 안전성 검증
    // Integer Box에서 꺼낸 데이터는 Integer 타입이므로, 바로 연산 가능
    int result = intContent * 2;
    IO.println("데이터를 이용한 연산 결과 (42 * 2): " + result);

    // 4. Raw Type (비권장)
    // Box rawBox = new Box(); 
    // rawBox.setContent(100); 
    // String content = (String) rawBox.getContent(); // 런타임 ClassCastException 위험!
}

```

## Generic 메서드 

```java
// E라는 타입 매개변수를 받는 Generic 메소드
// E는 메소드가 호출될 때 전달되는 배열의 타입으로 결정됩니다.
<E> void printArray(E[] inputArray) {
    IO.print("배열 내용: [");
    // 향상된 for 루프를 사용하여 배열의 모든 요소를 순회
    for (E element : inputArray) {
        IO.print(element + ", ");
    }
    // 마지막 ", " 제거 및 줄바꿈
    IO.println("\b\b]");
}

void main() {
    IO.println("--- Generic 메소드 printArray 활용 ---");

    // 1. String 배열을 사용하여 메소드 호출
    String[] stringArray = {"Apple", "Banana", "Cherry"};
    IO.print("String 배열: ");
    // 컴파일러가 E를 String으로 추론합니다.
    printArray(stringArray);

    // 2. Integer 배열을 사용하여 메소드 호출
    Integer[] integerArray = {10, 20, 30, 40};
    IO.print("Integer 배열: ");
    // 컴파일러가 E를 Integer로 추론합니다.
    printArray(integerArray);

    // 3. Double 배열을 사용하여 메소드 호출
    Double[] doubleArray = {1.1, 2.2, 3.3, 4.4};
    IO.print("Double 배열: ");
    // 컴파일러가 E를 Double로 추론합니다.
    printArray(doubleArray);
}

```
