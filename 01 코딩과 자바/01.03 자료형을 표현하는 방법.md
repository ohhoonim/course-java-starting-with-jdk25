# 01.03 자료형을 표현하는 방법

자료형은 ‘원시(Primitive)’ 타입과 ‘참조(Reference)’ 타입으로 구분됩니다. 타입은 자료형이라고도 부르며 보통 혼용해서 많이들 사용합니다. 원시 자료형은 ‘변수’항목에서 다뤘으므로 여기서는 ‘참조형’에 대해 자세히 알아봅니다.

참조 타입은 여러 종류의 원시, 참조 타입이 그룹으로 묶인 형태라고 생각하면 됩니다. 우선 class 부분을 살펴보면 이해하는데 어렵지 않습니다.

## class

class는 자바에서 참조 자료형을 표현하는 키워드입니다. class는 ‘필드’와 ‘메서드’로 구성되며, 기본 사용법은 아래와 같습니다. (자세한 사용법은 ‘실습’을 통해 연습할 예정입니다)
```java
// class를 표현하는 방법
class User {
	String name;
	int age;
	boolean isChildren;	

	String getName() {
		return name;
	}
	
	void setName(String name) {
		this.name = name;
	}
}
```

## 필드

‘name’, ‘age’, ‘isChidren’ 은 단순히 변수의 값을 담는 역할만 합니다. 이를 ‘필드’라고 합니다. 

> [자료형} + [변수명] 

이와 같은 형식으로 적어줍니다. 

## 메서드

‘getName()’ 은 ‘name’ 필드의 값을 던져주는 행위를 합니다. 이를 ‘매서드’라고 합니다. 타 언어에서는 ‘함수’라고도 부릅니다. 

```
[반환 자료형] + [메소드명] + ( [파라미터] … ) {
	// 로직
	return [반환할 값];
}
```

- [반환할 값]은 [반환 자료형]과 자료형이 같아야 합니다.
- [파라미터]는 여러 개를 사용할 수 있으며, 
- 각 파라미터는 [자료형] + [변수명] 형태로 적어주어야 합니다
- 값을 반활 할 때는 ‘return’ 키워드를 사용합니다.
- 반환할 값이 없을 경우에는 [반환 자료형]에 ‘void’를 적어줍니다

## enum

enum은 상수를 참조 자료형으로 표현하는 방법입니다. class 대신 enum을 사용합니다. 주로 필드를 대문자로 적어주는 것이 관례입니다. 

```java
// enum을 표현하는 방법
enum Grade {
	LEVEL1, LEVEL2, LEVEL3;
}
```

- Grade.LEVEL1 과 같이 사용합니다.

## record

record는 class내부의 필드 값들이 한 번 정해지면 변경이 불가능하게 만들어주는 class입니다.  

```java
record User (
	String name, int age, boolean isChildren
){}
```

## 자료형을 초기화하는 방법

클래스는 단순히 설계된 코드일 뿐이므로 이것을 메모리에 등록해야 사용할 수 있습니다. 이것을 인스턴스화 한다고 하는데 자바에서는 이때 ‘_new_’ 키워드를 사용하며, 이것 또한 연산자로 취급됩니다. 

```java
String name = new String(“Emma”);
```

조금 더 복잡한 애플리케이션을 코딩하게 되면 인스턴스를 생성하는 역할을 담당하는 factory 메소드를 활용하는 경우가 많습니다.

```java
void main() {
	User user = UserFactory.createUser(“Emma”);
}

class UserFactory {
	static User createUser(String name) {
		return new User(name);
	}
|

record User(String name) {}
```

## 인터페이스

인터페이스는 메서드를 정의만하고 구현하지 않습니다. 

```java
interface UserService {
	User findUserById(String email);
}
```

구현은 _implements_ 라는 키워드를 이용하여 class, record 에서 구현합니다.

```java
class UserServiceImpl implements UserService {
	@Override
	User findUserById(String email) {
		return new User(email);
	}
}
// 또는
record User (String name, String email) implements UserService {
	@Override
	User findUserById(String email) {
		return email;
	}
}
```

인터페이스가 왜 필요한지는 현시점에서 이해하기 힘드므로 사용법만 숙지하기 바랍니다.  interface 에서도 메서드를 구현할 수 있는 방법이 있는데, 메서드 앞에 default 키워드를 사용하면 됩니다.

```java
interface UserService {
	User findUserById(String email);

	default String getName() {
		return “Emma”;
	}
}
```

default를 사용하는 이유는 이미 인터페이스를 구현한 기존 클래스들이 많은 경우 유용하게 활용할 수 있습니다. default외에 static 키워드를 이용해도 구현할 수 있습니다. 

## 함수형 인터페이스와 람다식

Java는 ‘객체지향프로그래밍(OOP)’라는 프로그래밍 기법을 사용하는 대표적인 언어이지만 JDK8부터 ‘함수형프로그래밍(FP)’ 기법을 활용 할 수 있게 되었습니다. 여기서는 문법만 살펴보도록 하겠습니다. 

```java
void main() {
    Function<Integer, String> currency= (money) -> money + "원";
    IO.println(currency.apply(1000));
}
```

함수가 실행되는 형태를 “() -> {}” 와 같이 표현했는데 이를 ‘람다식’ 또는 ‘화살표 함수’라고 부릅니다. 자세한 “함수형프로그래밍” 기법은 여기서 다루지 않습니다. 
람다식을 사용할 수 있으려면 메소드가 하나만 선언되어있는 인터페이스를 작성하면 됩니다. 

```java
@FunctionalInterface
interface PlusFunction {
	int plus(int a, int b);
}

// 람다식으로 표현
void main() {
	PlusFunction add = (a, b) -> a + b + 1;
	IO.println(add.plus(1, 2));
}
```

## 어노테이션

앞선 내용 중에 ‘@Override’, ‘@FunctionalInterface’ 를 볼 수 있었는데 ‘@‘가 붙은 것들을 어노테이션이라고 합니다. 애플리케이션은 컴파일 되거나 실행되는 시점에 특정 정보를 획득할 필요가 있는데 이를 ‘메타데이터’라고 합니다. 

메타데이터로서의 어노테이션은 여러 방면으로 광범위하게 활용되고 있으므로 새로운 어노테이션을 발견할 때마다 지나치지말고 그 용도를 익혀두어야 합니다.

## 패키지와 모듈

패키지는 자바 코드 파일을 논리적으로 그룹화하는 단위이다. 파일에서 폴더 구조와 비교해보면,

```java
com.ohhoonim.component

com
	ㄴohhoonim
		ㄴcomponent
```

패키지를 사용하면 class의 이름이 충돌하는 것을 방지해줍니다. java.util.List와 java.awt.List

패키지 사용법은 작성한 자바파일의 최상단에 다음과 같이 적어주면 됩니다.

```java
// 파일: com/mycompany/app/Logger.java
package com.mycompany.app;

public class Logger { /* ... */ }
```

모듈은 JDK9부터 도입된 개념으로 패키지보다 더 큰 규모의 코드단위입니다. 실무에서는 아직까지도 잘 사용되지 않고 있습니다.

```java
// 파일: module-info.java

module com.mycompany.service {
    // 이 모듈이 외부에 공개할 패키지를 명시
    exports com.mycompany.service.api; 
    
    // 이 모듈이 사용(의존)할 다른 모듈을 명시
    requires java.sql; 
}
```

## 접근 한정자

클래스, 필드, 메소드, 생성자에 대한 접근 범위를 제어하는 키워드가 총 3개가 존재합니다. (종류는 4가지 입니다)

- public : 어디서든 이용 가능
- protected : 상속관계인 경우 이용가능 (extends, implements)
- private : 클래스 내에서만 이용가능 
- 접근한정자 키워드를 사용하지 않은 경우 : 동일 패키지 내에서만 접근 가능

## [주의] 이번 장에서 설명하지 않은 기능

(앞으로도 설명할 생각이 없는 기능)

이번 장에서 ‘class 상속’ 기능은 설명하지 않았습니다. interface를 구현하려면 ‘implements’ 키워드를 사용하듯이 부모 class의 기능을 자식 class가 사용가능하도록 하려면 ‘_extends_’ 키워드를 사용합니다. 접근한정자 protected는 상속과 관련이 있습니다.
