# 01.04 프로그래밍 실행 흐름 제어

애플리케이션 <u>실행의 흐름을 ‘로직(Logic)’</u>이라고 합니다. 로직을 처리하기 위해 <u>‘어떤 상황에서 무엇을 할지를 결정’하는 문법들이 존재하는데 이를 제어문</u>이라고 하며, 프로그래밍에서 로직을 처리하는 중요한 부분이므로 <u>알고리즘 풀이를 통해 많은 연습이 필요</u>합니다.

## 조건문 : if ~ else

```java
void main() {
   
   int score = 85;
   String grade;

   if (score >= 90) { // 90점 이상인 경우
       grade = "A";
   } else if (score >= 80) { // 90점 미만, 80점 이상인 경우
       grade = "B";
   } else if (score >= 70) { // 80점 미만, 70점 이상인 경우
       grade = "C";
   } else { // 70점 미만인 경우 (나머지)
       grade = "F";
   }

   IO.println("당신의 점수는 " + score + "점이며, 등급은 " + grade + "입니다.");
   
}
```

코드가 작성된 if문 순서대로 진행되고, 조건이 만족하면 if문을 빠져나간 후 println() 부분을 실행하게된다.

## [참고] 괄호의 중첩과 변수의 Scope

괄호는 중첩되어 표현될 수 있다. 이 때 바깥 괄호에 선언된 변수는 안쪽 괄호에서 사용할 수 있지만 그 반대는 허용되지 않는다. 위 예제에서 ‘grade’ 변수가 바깥 괄호에 선언되었으므로 if 문 안쪽 괄호에서 ‘grade’를 사용할 수 있다.

## 반복문 : for

```java
public class LoopExample {
    public static void main(String[] args) {
        int sumFor = 0; // 합계를 저장할 변수 초기화

        // 초기값(i=1), 반복 조건(i<=100), 증감식(i++)
        for (int i = 1; i <= 100; i++) {
            sumFor = sumFor + i; // sumFor += i; 와 동일
        }

        System.out.println("for 반복문 결과 (1~100 합계): " + sumFor);
    }
}
```

## 반복문 : while

```java
public class LoopExample {
    public static void main(String[] args) {
        int sumWhile = 0; // 합계를 저장할 변수 초기화
        int i = 1;      // 💡 초기화는 while문 시작 전에 명시

        // 반복 조건(i <= 100)
        while (i <= 100) {
            sumWhile = sumWhile + i; // sumWhile += i; 와 동일
            
            i++; // 💡 증감식은 while문 내부에 명시
        }

        System.out.println("while 반복문 결과 (1~100 합계): " + sumWhile);
    }
}
```

## [참고] for와 while 반복문을 사용하는 적절한 상황

- for : 정확한 횟수 반복, 배열 순회, 수열 증감
- while : 조건 기반 반복, 무한 루프, (파일)데이터 처리, 재귀적 알고리즘 

## 예외 처리

자바에서 예외는 두 가지로 구분됩니다.

- Error : 애플리케이션이 처리할 수 없는 문제. StackOverflow, OutOfMemory 등
- Exception : 애플리케이션이 로직으로 처리할 수 있는 문제.

일반적으로 Java에서의 예외처리는 Exception을 말하며 예외 발생시 try~catch 문을 사용하여 

- 예외사황을 복구하거나,
- 또 다른 예외를 발생시킨다

```java
// try~catch 기본 사용법
void main() {

	try {
		int result = 10 / 0;
	} catch (Exception e) {
		IO.println(e.getMessage());   // ‘/ by zero’
	}
}
```

예외가 발생하면 try~catch로 ‘후 처리’를 하는 것이 원칙이지만, Exception 대신에 RuntimeException을 상속받은 예외들은 ‘후 처리’를 하지 않아도 되는데 이것을 Unchecked Exception 이라고 합니다. 

주로 시스템, 파일 입출력, 네트워크와 관련된 예외는 Exception으로 처리를 하고, 그 외 대부분의 경우 RuntimeException으로 처리를 해줍니다. 

예외처리에 대한 깊은 내용은 ‘파일 입출력과 스레드’에서 다룹니다.

## 조건문 : switch

if 조건문의 또다른 형태이다. 다음에 설명할 패턴매칭과 결합하면 if 문보다 훨씬 편리하다.

```java
void main() {
  int dayOfWeek = 3; // 1:월요일, 2:화요일, ..., 7:일요일
	String week = switch (dayOfWeek) {
		case 1 -> "월";
		case 2 -> "화";
		case 3 -> "수";
		case 4 -> "목";
		case 5 -> "금";
		case 6 -> "토";
		case 7 -> "일";
     default -> "없는";
	};
	IO.println("오늘은 " + week + "요일 입니다");
}
```

## 패턴매칭

패턴 매칭(Pattern Matching)은 프로그래밍에서 사용되는 개념으로 <u>‘데이터 구조를 검사(Matching)’하는 동시에 해당 구조의 ‘구성 요소(패턴)’를 추출하여 변수에 바인딩하는 기법</u>입니다. 

```java
// instanceof 패턴매칭
void main() {
	var user = “Emma”;

	if (user instanceof String s) { // 타입 확인 + s 변수 바인딩 
			IO.println(“name: “ + s); // s 사용 
	}
}
```

```java
// switch 패턴매칭
void main() {
	Object obj = “Emma”;

   String description = switch (obj) {
       // obj가 Integer 타입이면 intValue 변수로 바인딩
       case Integer intValue -> "정수 값: " + intValue; 
       
       // obj가 String 타입이면 s 변수로 바인딩
       case String s -> "문자열 길이: " + s.length(); 
        
       // null 값 처리 (JDK 21 이전에는 switch에서 null 처리 불가)
       case null -> "값 없음 (null)"; 
        
       // 나머지 모든 경우를 처리
       default -> "알 수 없는 타입";
    };

    IO.println(description);  // “문자열 길이: 4”
}
```

```java
// 레코드 패턴 매칭
void main() {
	Object point = new Point(10, 20);

		IO.println(“Point 객체입니다. 좌표: (“ + px + “, “ + py +”)”);
	}
}

record Point(int x, int y) {}
```
