# 02.01 파일 입출력

## 패스 (Path)

기본적으로 모든 OS는 실행파일(.exe, .sh 등)은 자신이 포함된 폴더 내에서만 실행가능하고 외부 파일 접근도 마찬가지입니다. 파일 실행을 폴더 위치에 상관없이 실행하려면  각 OS에 맞게 PATH 설정을 해주어야 합니다. 

```sh
C:\Program Files\pdf-viewer.exe
# pdf-viewer.exe는 myReport.pdf 라는 파일에 임의로 접근할 수 없다 
# 프로그램을 실행시킨 후 '파일열기'를 해야하는 이유이다
C:\Users\user\Documents\myReport.pdf
```

> 📋 Windows와 Linux/Macos에서의 경로 구분자
> - Windonws : 역슬래시 `\` 
> - Linux/Macos : 슬래시 `/`
> - 코딩할 때는 일반적으로 '/'를 사용하면 됩니다

## 클래스패스 (classpath)

Java에서도 PATH의 제약을 받으므로 path 설정이 필요합니다. Java에서는 주로 컴파일된 .class 파일, 외부라이브러리(.jar)의 참조경로가 필요한데 이를 classpath라고 한다 

```sh
# 클래스패스를 지정하는 방법
$ java -cp ".;lib/mylib.jar" com.example.MainClass
```

## File 읽고 쓰기

Java에서 Files와 Paths를 사용하면 파일을 다루는 것은 어렵지 않습니다. 

```java
void main() {
    // 1. 파일 경로 지정 (현재 실행 디렉토리에 파일 생성)
    Path filePath = Paths.get("simple_example.txt");
    String contentToWrite = "Hello, Java File I/O!\nThis is the second line.";

    IO.println("--- 파일 쓰기 시작 ---");

    try {
        // 2. 파일 쓰기: 문자열을 파일에 한 번에 씁니다.
        Files.writeString(filePath, contentToWrite);
        IO.println("파일 쓰기 완료: " + filePath.toAbsolutePath());

        IO.println("\n--- 파일 읽기 시작 ---");

        // 3. 파일 읽기: 파일 전체 내용을 문자열로 한 번에 읽습니다.
        String contentRead = Files.readString(filePath);

        IO.println("파일 내용:\n" + contentRead);

    } catch (IOException e) {
        // 4. 예외 처리: 파일 입출력 오류 발생 시 처리
        IO.println("파일 입출력 오류가 발생했습니다: " + e.getMessage());
        e.printStackTrace();
    }
}
```

## InputStream과 OutputStream

자바에서는 데이터의 흐름을 stream 이라고 합니다. 들어오는(입력받는) 스트림을 InputStream, 나가는(출력하는) 스트림을 OutputStream 이라고 합니다. 스트림의 특징에 따라 크게 다음 두가지로 구분합니다. 

- 바이트 스트림 : 이미지, 오디오, 비디오 등의 바이너리 데이터
    - InputStream / OutputString
- 문자 스트림 : 텍스트파일, 네트워크 문자열 등의 문제 데이터 처리
    - Reader / Writer

상세한 내용은 '필수 디자인 패턴 - Decorator 패턴' 에서 다루겠습니다. 

## try-with-resource문

파일 입출력 스트림, 데이터베이스 연결, 소켓 등 사용 후 반드시 닫아야 하는 자원(Resource)을 보다 안전하고 간결하게 처리하기 위해 사용됩니다.

이 구문을 사용하면 <u>자원 해제를 위한 finally 블록이 필요 없으며,</u> 예외 발생 유무와 관계없이 자원이 자동으로 해제되는 것을 보장합니다.

try-with-resource문에서 사용할 자원은 java.lang.AutoCloseable 인터페이스를 구현해야 합니다.

```java 
class TWRDemo {
    public void readFirstLine(String filePath) {
        
        // 괄호 () 안에 자원(FileReader, BufferedReader)을 선언합니다.
        try (
            FileReader fr = new new FileReader(filePath);
            BufferedReader br = new BufferedReader(fr) // 자원을 여러 개 선언할 수 있음
        ) {
            
            // 파일 읽기 로직 (자원 사용)
            String line = br.readLine();
            System.out.println("첫 번째 줄: " + line);

        } catch (IOException e) {
            // 파일이 없거나 읽기 오류 발생 시 처리
            System.err.println("오류 발생: " + e.getMessage());
        } 
        // finally 블록 없이 자원(br, fr)이 자동으로 닫힙니다.
    }
}
```
