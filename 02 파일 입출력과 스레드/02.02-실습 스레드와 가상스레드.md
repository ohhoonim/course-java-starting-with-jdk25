# 02.02 스레드와 가상스레드

## 실습

### 실습 1) run() 을 사용하지 않고 start()를 사용하는 이유

> 📋 start()와 run()의 차이
> - run() 메소드: 스레드가 실제로 수행할 작업을 정의하는 부분입니다. 이 메소드를 직접 호출하면, 스레드가 아닌 현재 스레드(메인 스레드) 내에서 단순한 메소드 호출로 실행됩니다.
> - start() 메소드: 새로운 스레드를 생성하고, OS에 이 스레드를 실행해 달라고 요청합니다. 이후 JVM이 생성된 스레드의 run() 메소드를 호출합니다. 반드시 start()를 호출해야 멀티스레딩이 이루어집니다.

### 실습 2) Thread Pool 

- Thread Pool 을 사용하여 스레드를 구동하는 코르를 작성하시오

```java
class WorkerTask implements Runnable {
    private int taskId;

    public WorkerTask(int taskId) {
        this.taskId = taskId;
    }

    @Override
    public void run() {
        // 현재 작업을 수행하는 스레드의 이름 출력
        IO.println("작업 " + taskId + " 시작. (스레드: " + Thread.currentThread().getName() + ")");
        try {
            // 작업 수행 시간 모방
            Thread.sleep(200);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
        IO.println("작업 " + taskId + " 완료.");
    }
}

// JDK 25의 간소화된 main 메소드
void main() {
    // 1. 고정된 크기(3개)의 스레드 풀 생성 (ExecutorService)
    // 이 스레드들은 작업이 끝나도 파괴되지 않고 재사용을 위해 풀에 대기합니다.
    ExecutorService executorService = Executors.newFixedThreadPool(3);

    // 2. 5개의 작업을 스레드 풀에 제출합니다. (작업 큐에 저장됨)
    for (int i = 1; i <= 5; i++) {
        Runnable worker = new WorkerTask(i);
        executorService.execute(worker); // 스레드 풀에 작업 제출
    }

    // 3. 더 이상 새로운 작업을 받지 않도록 셧다운 요청
    executorService.shutdown();

    // 4. 모든 작업이 완료될 때까지 최대 5초 대기
    try {
        if (executorService.awaitTermination(5, TimeUnit.SECONDS)) {
            IO.println("\n모든 작업이 성공적으로 완료되었습니다.");
        } else {
            IO.println("\n시간 초과. 일부 작업이 완료되지 않았을 수 있습니다.");
        }
    } catch (InterruptedException e) {
        IO.println("메인 스레드 대기 중 중단됨.");
        Thread.currentThread().interrupt();
    }
}
```


### 실습 3) Executors가 제공하는 팩토리 메소드를 조사하시오

- newFixedThreadPool(int nThreads): 고정된 개수의 스레드 풀을 생성합니다.
- newCachedThreadPool(): 필요할 때만 새로운 스레드를 생성하고, 사용되지 않는 스레드는 60초 후 자동으로 제거됩니다.
- newSingleThreadExecutor(): 하나의 스레드만 사용하여 작업을 순차적으로 처리합니다
- newScheduledThreadPool(int corePoolSize): 특정 시간 지연 후 또는 주기적으로 반복되는 작업을 예약 실행할 수 있는 풀을 생성합니다.

### 실습 4) ForkJoinPool

- ForkJoinPool은 대용량의 작업을 재귀적으로 작은 서브태스크로 나누고(Fork), 결과를 합치는(Join) 분할 정복(Divide and Conquer) 알고리즘에 최적화된 특수 목적의 스레드 풀입니다.
- 예시 코드를 보고 모르는 부분은 AI 챗을 활용하여 이해하시오. 
- 분할 정복 알고리즘에 대해 조사하시오

```java

class SumTask extends RecursiveTask<Long> {
    // 💡 작업을 나누는 최소 임계값 (Threshold)
    private static final int THRESHOLD = 100_000;

    private final long start;
    private final long end;
    private final long[] array;

    public SumTask(long[] array, long start, long end) {
        this.array = array;
        this.start = start;
        this.end = end;
    }

    @Override
    protected Long compute() {
        long length = end - start;

        // 1. 멈춤 조건 (Base Case): 작업 크기가 임계값보다 작으면 직접 계산
        if (length <= THRESHOLD) {
            return computeSequentially();
        }

        // 2. 분할 (Fork): 작업을 두 개의 서브 태스크로 나눕니다.
        long mid = (start + end) / 2;

        SumTask leftTask = new SumTask(array, start, mid);
        SumTask rightTask = new SumTask(array, mid, end);

        // 왼쪽 작업을 현재 스레드에서 실행 요청 (비동기)
        leftTask.fork();

        // 오른쪽 작업을 현재 스레드에서 실행 요청 (동기 또는 재귀 호출)
        Long rightResult = rightTask.compute();

        // 3. 결합 (Join): 왼쪽 작업의 결과를 기다려 합산합니다.
        Long leftResult = leftTask.join();

        return leftResult + rightResult;
    }

    // 임계값보다 작을 때 일반 for 루프로 합산하는 메소드
    private Long computeSequentially() {
        long sum = 0;
        for (long i = start; i < end; i++) {
            sum += array[(int) i];
        }
        // IO.println("계산: " + start + " ~ " + end); // 디버깅용
        return sum;
    }
}

void main() {
    // 1. 대규모 배열 생성 (1부터 10,000,000까지)
    int SIZE = 10_000_000;
    long[] numbers = new long[SIZE];
    for (int i = 0; i < SIZE; i++) {
        numbers[i] = i + 1;
    }

    // 2. ForkJoinPool 생성 (일반적으로 시스템의 프로세서 코어 수에 맞게 생성됨)
    ForkJoinPool pool = new ForkJoinPool();

    // 3. 최초의 메인 작업을 생성합니다.
    SumTask mainTask = new SumTask(numbers, 0, SIZE);

    // 4. 작업을 풀에 제출하고 결과를 기다립니다.
    long startTime = System.currentTimeMillis();

    // invoke()는 작업을 제출하고 완료될 때까지 기다린 후 결과를 반환합니다.
    long result = pool.invoke(mainTask);

    long endTime = System.currentTimeMillis();

    // 5. 결과 및 시간 출력
    // 1부터 N까지의 합 공식: N * (N + 1) / 2
    long expectedResult = (long) SIZE * (SIZE + 1) / 2;

    IO.println("배열 크기: " + SIZE);
    IO.println("예상되는 합계: " + expectedResult);
    IO.println("병렬 계산 결과: " + result);
    IO.println("계산 소요 시간: " + (endTime - startTime) + " ms");

    // 6. 풀 종료 (ExecutorService와 달리 shutdown()이 필수는 아니지만 명시적으로 닫는 것이 좋습니다.)
    pool.shutdown();
}

```

> 💡 예제 동작 원리
> - 초기 작업 제출: mainTask가 ForkJoinPool에 제출됩니다.
> - 분할: compute() 메소드가 실행되어 작업의 크기(length)가 THRESHOLD(100,000)보다 크므로, 작업을 leftTask와 rightTask로 나눕니다.
> - fork() 호출: leftTask.fork()는 왼쪽 작업을 풀의 큐에 넣고 비동기적으로 실행합니다. 현재 스레드는 오른쪽 작업으로 넘어갑니다.
> - compute() 호출: rightTask.compute()는 현재 스레드에서 오른쪽 작업을 동기적으로 처리하거나, 필요하면 다시 재귀적으로 분할합니다.
> - join() 호출: leftTask.join()은 비동기적으로 실행된 왼쪽 작업이 완료될 때까지 기다립니다. 만약 왼쪽 작업이 아직 완료되지 않았다면, 현재 스레드는 다른 작업(work-stealing)을 수행하여 CPU 유휴 시간을 줄이려고 시도합니다.
> - 결합: 왼쪽과 오른쪽의 결과가 모두 준비되면 합산하여 반환합니다. 이 과정이 재귀적으로 반복되어 최종 결과를 얻게 됩니다.

### 실습 5) 가상스레드 vs 플랫폼 스레드

- 가상 스레드와 플랫폼 스레드의 차이점을 조사하시오.
- Scoped Value에 대하여 조사하시오.
- 가상스레드를 사용할 때 스레드 풀 방식으로 사용하지 말것을 권장하는데 그 이유를 조사하시오.

> 📋 가상 스레드는 항상 플랫폼 스레드(Carrier Thread) 위에서 실행됩니다.
> 
> - 가상 스레드는 작업을 실행하다가 파일 I/O나 네트워크 호출처럼 블로킹(Blocking)이 발생하는 순간, 스스로를 플랫폼 스레드에서 분리(Unmount)시키고 작업을 멈춥니다.
> - 이때 플랫폼 스레드는 다른 대기 중인 가상 스레드를 실행하는 데 재활용됩니다.
> - 블로킹이 해제되면, 가상 스레드는 다시 풀에 있는 사용 가능한 플랫폼 스레드에 탑재(Mount)되어 중단된 곳부터 실행을 재개합니다.
> 
> 이러한 방식으로, 적은 수의 커널 스레드를 사용하여 수십만 개의 가상 스레드를 생성하고 동시에 관리할 수 있습니다.
