# 02.02 스레드와 가상스레드

## 스레드

스레드(Thread)는 프로세스(Process) 내에서 실행되는 가장 작은 단위입니다. 하나의 프로세스는 하나 이상의 스레드를 가질 수 있으며, 이 스레드들은 프로세스의 메모리 공간을 공유하면서 동시에 작업을 수행할 수 있습니다. 자바에서 스레드를 사용하는 것은 멀티태스킹(Multi-tasking), 즉 여러 작업을 병렬적(concurrently)으로 처리하여 애플리케이션의 응답성과 효율성을 높이는 핵심 방법입니다.

- 동시성 (Concurrency): 여러 스레드가 CPU 시간을 나누어 사용하며 동시에 실행되는 것처럼 보이게 합니다.
- 자원 공유: 같은 프로세스 내의 스레드들은 힙(Heap) 영역의 메모리를 공유합니다. 이로 인해 데이터를 쉽게 주고받을 수 있지만, 동기화(Synchronization) 문제를 발생시킬 수 있습니다.
- 생성 방법: 자바에서 스레드를 생성하는 방법은 크게 두 가지입니다.
    - Thread 클래스 상속: Thread 클래스를 상속받아 run() 메소드를 오버라이드합니다.
    - Runnable 인터페이스 구현: Runnable 인터페이스를 구현하여 run() 메소드를 정의합니다. (더 권장되는 방식입니다.)

```java
class MyRunnable implements Runnable {
    private String name;

    public MyRunnable(String name) {
        this.name = name;
    }

    @Override
    public void run() {
        for (int i = 0; i < 3; i++) {
            IO.println(name + " 실행 중: " + i);
            try {
                // 0.1초 대기 (스레드 전환을 확인하기 위함)
                Thread.sleep(100);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
        IO.println(name + " 종료.");
    }
}

// JDK 25의 간소화된 main 메소드
void main() {
    IO.println("메인 스레드 시작.");

    // 1. Runnable 객체 생성
    Runnable task1 = new MyRunnable("스레드-A");
    Runnable task2 = new MyRunnable("스레드-B");

    // 2. Thread 객체를 생성자에 Runnable 객체를 전달하여 생성
    Thread threadA = new Thread(task1);
    Thread threadB = new Thread(task2);

    // 3. 스레드 시작 (OS에 실행 요청)
    threadA.start();
    threadB.start();

    // 메인 스레드는 두 스레드와 별개로 계속 실행됩니다.
    IO.println("메인 스레드 종료.");
}
```

## 스레드 풀 
스레드 풀(Thread Pool)은 작업을 처리하기 위해 미리 생성해 둔 스레드들의 집합입니다. 작업 요청이 들어올 때마다 새로운 스레드를 생성하는 대신, 풀에 있는 스레드를 재사용하여 성능 오버헤드를 줄이고 자원을 효율적으로 관리합니다.

- 성능 향상: 스레드를 생성하고 소멸하는 비용을 줄입니다 
- 자원 관리: 최대 스레드 수를 제한하여, 시스템 자원이 고갈되는 것을 방지합니다. 
- 작업 관리 용이: 스레드 생성, 실행, 종료 로직에 신경 쓸 필요가 없습니다. 

## 가상스레드

가상 스레드는 기존의 자바 스레드(플랫폼 스레드)가 가진 문제점을 해결하고, 고성능의 동시성 애플리케이션을 더 쉽게 개발할 수 있도록 합니다. 가상스레드는 플랫폼 스레드 위에서 실행됩니다.

```java

void main() {
    IO.println("가상 스레드 시작...");

    // 1. 가상 스레드 생성 (이것은 스레드 풀의 인터페이스를 사용하지만 풀링을 하지 않는다. )
    ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor();

    // 2. 10개의 간단한 작업을 제출합니다.
    IntStream.range(0, 10).forEach(i -> {
        executor.submit(() -> {
            // 현재 작업이 어떤 스레드에서 실행되는지 확인
            String threadName = Thread.currentThread().getName();

            IO.println("작업 " + i + " 실행: " + threadName);

            // I/O 블로킹 상황을 모방 (이때 캐리어 스레드가 해제됨)
            try {
                // Thread.sleep(1)은 1밀리초를 의미합니다.
                Thread.sleep(10000);
            } catch (InterruptedException e) {
                IO.println("i/o 블록킹");
                Thread.currentThread().interrupt();
            }
        });
    });

    // 3. 작업 제출을 중단 (새로운 작업 거부)
    executor.shutdown();

    System.out.println("==============");

    // 4. 모든 작업이 완료될 때까지 메인 스레드가 대기합니다.
    try {
        // 모든 작업이 완료되거나, 최대 5초가 경과될 때까지 대기
        if (executor.awaitTermination(5, TimeUnit.SECONDS)) {
            IO.println("모든 가상 스레드 작업 완료.");
        } else {
            IO.println("시간 초과: 일부 작업이 5초 내에 완료되지 않았습니다.");
        }
    } catch (InterruptedException e) {
        IO.println("메인 스레드 대기 중 중단됨.");
        Thread.currentThread().interrupt();
    }
}
```
