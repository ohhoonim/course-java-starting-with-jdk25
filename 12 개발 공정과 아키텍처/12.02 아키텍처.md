# 아키텍처

아키텍처(Architecture)는 소프트웨어 시스템, 건물, 또는 조직과 같은 복잡한 구조체의 기본 구성 요소(Elements), 이들 간의 관계(Relationships), 그리고 이들을 설계하고 진화시키는 원칙과 가이드라인을 정의하는 총체적인 개념입니다.

쉽게 말해, 아키텍처는 시스템의 큰 그림을 그린 설계도이며, '무엇을(What)', '어떻게(How)', '왜(Why)' 그렇게 구성했는지에 대한 결정의 집합입니다.

## 소프트웨어 아키텍처의 핵심

소프트웨어 아키텍처는 시스템이 충족해야 하는 기능적 요구사항뿐만 아니라, 비기능적 요구사항(품질 속성)을 달성하는 데 중점을 둡니다.

### 구성 요소 (Elements)
시스템을 이루는 주요 모듈, 컴포넌트, 서비스, 데이터베이스, 사용자 인터페이스 등입니다. (예: `UserService`, `PaymentGateway`, `Database`)

## 관계 (Relationships)
구성 요소들이 서로 어떻게 상호작용하고 통신하는지 정의합니다. (예: `REST API 호출`, `메시지 큐 통신`, `데이터베이스 연결`)

### 원칙 (Principles)
아키텍처를 설계할 때 따르는 규칙과 제약 조건입니다. (예: `보안성`, `확장성`, `유지보수성`)

## 아키텍처의 중요성 (품질 속성)

훌륭한 아키텍처는 시스템의 품질 속성(Non-Functional Requirements)을 보장하는 기반이 됩니다.

* 확장성 (Scalability): 사용자 증가나 부하 증가에 대비하여 시스템을 쉽게 확장할 수 있는 능력.
* 성능 (Performance): 시스템이 작업을 효율적으로 처리하는 속도와 응답 시간.
* 유지보수성 (Maintainability): 시스템을 쉽게 수정, 개선, 디버깅할 수 있는 능력.
* 신뢰성 (Reliability): 시스템이 오류 없이 의도한 대로 일관되게 작동하는 능력.
* 보안성 (Security): 시스템과 데이터가 무단 접근으로부터 보호되는 정도.

## 대표적인 아키텍처 패턴

소프트웨어 시스템을 조직하는 데 널리 사용되는 몇 가지 주요 패턴입니다.

| 패턴                                    | 설명                                                                                                   | 적합한 상황                                                       |
| :-------------------------------------- | :----------------------------------------------------------------------------------------------------- | :---------------------------------------------------------------- |
| 단일형 아키텍처 (Monolithic)            | 모든 구성 요소가 단일 코드베이스와 단일 서비스로 묶여 배포되는 전통적인 방식.                          | 소규모 프로젝트, 단순한 애플리케이션, 빠른 초기 개발이 필요할 때. |
| 계층형 아키텍처 (Layered, N-Tier)       | 시스템을 역할별로 수평적인 계층(프레젠테이션, 비즈니스 로직, 데이터 접근)으로 나누는 방식. (가장 흔함) | 엔터프라이즈 애플리케이션, 내부 비즈니스 로직이 명확할 때.        |
| 마이크로서비스 아키텍처 (Microservices) | 시스템을 독립적으로 배포 가능한 작고 느슨하게 결합된 서비스들의 집합으로 구성하는 방식.                | 대규모, 고도화된 확장성이 필요한 서비스(넷플릭스, 아마존 등).     |
| 이벤트 중심 아키텍처 (Event-Driven)     | 컴포넌트들이 이벤트를 발행하고 구독함으로써 비동기적으로 통신하는 방식.                                | 실시간 처리, 복잡한 데이터 파이프라인, 낮은 지연시간이 필요할 때. |
