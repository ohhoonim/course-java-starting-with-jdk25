# url로 접근 제어하기

## Authorization 

Authorization (인가)는 Spring Security를 포함한 모든 보안 시스템에서 인증(Authentication)이 완료된 사용자에게 특정 자원(Resource)이나 기능에 접근할 수 있는 권한을 부여하는 과정을 의미합니다.

쉽게 말해, "당신은 누구인지 확인했고(인증), 이제 당신이 무엇을 할 수 있는지 결정하는 것(인가)"입니다.

인가 과정은 주로 다음 세 가지 핵심 요소를 기반으로 이루어집니다.

- Principal (주체)
  : 자원에 접근하려는 사용자 또는 프로세스입니다. Spring Security에서는 인증을 통해 확보된 UserDetails 객체로 표현됩니다.
- Resource (자원)
  : 접근을 시도하는 대상입니다. 웹 애플리케이션에서는 URL 경로, HTTP 메소드, 특정 서비스 메소드, 데이터베이스 테이블 등이 될 수 있습니다.
- Authority / Role (권한/역할)
  : 주체에게 부여된 접근 능력의 집합입니다. Spring Security에서는 주로 GrantedAuthority 인터페이스를 구현한 Role (역할)을 사용합니다 (예: ROLE_ADMIN, ROLE_USER).

## Spring Security에서의 url 인가 예제

```java
http
    .authorizeHttpRequests(authorize -> authorize
        // 1. URL 패턴과 HTTP 메소드 지정
        .requestMatchers(HttpMethod.POST, "/posts").hasRole("WRITER") 
        
        // 2. 경로 기반 역할 지정
        .requestMatchers("/admin/**").hasAuthority("ROLE_ADMIN") 
        
        // 3. 나머지 모든 요청은 인증만 필요
        .anyRequest().authenticated() 
    );
```
