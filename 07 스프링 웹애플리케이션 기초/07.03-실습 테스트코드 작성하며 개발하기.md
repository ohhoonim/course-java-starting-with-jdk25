# 테스트코드 작성하며 개발하기

## 실습

### 실습 1) @Controller 슬라이스 테스트 

- 다음의 Post class 다이어그램을 참고하여 Post rest api를 작성하고 controller에 대한 테스트 코드를 작성하시오.
- 구현사항
  - 데이터베이스는 사용하지 않음
  - CRUD 를 모두 구현해야함
  - REST 규칙을 최대한 사용하시오. Method, URL

```mermaid
classDiagram
    direction LR

    class Post {
        +Long id
        +String title
        +String content
        +Date createdAt
        +List<Comment> comments
        +addComment(Comment comment)
        +removeComment(Comment comment)
    }

    class Comment {
        +Long id
        +String author
        +String content
        +Date createdAt
        +Post post
    }

    Post "1" -- "*" Comment : contains
```

| 구성 요소                          | 설명                                                                                                                                                                                                                                                                                                                  |
| ---------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Post 클래스                        | 게시글 정보를 나타냅니다. <br>- id, title, content, createdAt: 게시글의 기본 속성입니다.  <br>- comments: List<Comment>: 이 Post에 달린 Comment 객체들의 리스트를 가집니다. 이는 일대다 관계의 "다(Many)" 쪽을 참조하는 속성입니다.  <br>- addComment(), removeComment(): Comment를 추가하거나 제거하는 메소드입니다. |
| Comment 클래스                     | 댓글 정보를 나타냅니다.<br>- id, author, content, createdAt: 댓글의 기본 속성입니다.  <br>- post: Post: 이 Comment가 어떤 Post에 속하는지 참조하는 속성입니다. 이는 일대다 관계의 "일(One)" 쪽을 참조합니다.                                                                                                          |
| Post "1" -- "*" Comment : contains | Post와 Comment 간의 관계를 나타냅니다.  <br>- "1": Post는 Comment에 대해 "하나"의 관계를 가집니다.  <br>- "*": Comment는 Post에 대해 "다수"의 관계를 가집니다 (즉, 하나의 게시글에 여러 댓글이 달릴 수 있습니다).  <br>- : contains: 관계의 의미를 "포함하다"로 설명합니다.                                           |
