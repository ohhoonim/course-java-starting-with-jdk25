### 실습 2) Profile 

- Spring Profile은 Spring Framework 또는 Spring Boot 애플리케이션에서 환경에 따라 빈(Bean) 설정 및 구성 정보를 다르게 로드할 수 있게 해주는 기능입니다. 이를 통해 개발 환경(development), 테스트 환경(test), 운영 환경(production) 등 서로 다른 환경에 맞는 애플리케이션 설정을 손쉽게 전환할 수 있습니다.
- appolication.yml에서 Profile을 'dev', 'prod' 로 분리하시오 
    - dev : 8080 포트로 설정
    - prod : 9090 포트로 설정 


| 파일 유형            | 설명                                                      |
| -------------------- | --------------------------------------------------------- |
| application.yml      | 모든 환경에 공통으로 적용되는 기본 설정 (Default Profile) |
| application-dev.yml  | dev 프로파일이 활성화될 때 기본 설정을 재정의하는 파일    |
| application-prod.yml | prod 프로파일이 활성화될 때 기본 설정을 재정의하는 파일   |

```yaml
# application.yml
spring:
  application:
    name: my-profile-service
  # 모든 환경에 공통으로 적용될 기타 설정...
logging:
  level:
    root: INFO
```

```yml
# application-dev.yml
server:
  port: 8080 # 개발 환경 포트

spring:
  datasource:
    url: jdbc:h2:mem:devdb # 개발용 인메모리 DB
  jpa:
    show-sql: true # 쿼리 출력 활성화
```

```yml
# application-prod.yml
server:
  port: 9090 # 운영 환경 포트

spring:
  datasource:
    url: jdbc:mysql://prod-server:3306/realdb # 운영 DB
    username: realuser
    password: ${DB_PASSWORD} # 환경 변수 사용
  jpa:
    show-sql: false # 쿼리 출력 비활성화
```

```sh
# dev 프로파일을 활성화합니다.
java -jar myapp.jar --spring.profiles.active=dev
```

```sh
# 1. 환경 변수 설정 (DB_PASSWORD라는 이름으로 실제 비밀번호를 설정)
export DB_PASSWORD=my_secure_prod_password

# 2. 애플리케이션 실행 시 prod 프로파일 활성화
java -jar myapp.jar --spring.profiles.active=prod
```
