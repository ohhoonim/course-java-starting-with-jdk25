## 스프링 부트 Starter 제작 가이드 🛠️

Starter를 만들면 사용자는 단 하나의 의존성만 추가하여 해당 기능을 바로 사용할 수 있게 됩니다.

-----

### 1\. 🎯 Starter의 구성 요소 및 목적

스프링 부트 Starter 모듈은 주로 두 가지 핵심 구성 요소로 나뉩니다.

#### A. 자동 설정 모듈 (Auto-Configuration Module) - 핵심

이 모듈은 Starter가 실제로 기능을 제공하는 부분입니다.

  * **패키지 이름 규칙**: 보통 `[기술명]-spring-boot-autoconfigure` 형태를 따릅니다.
  * **역할**: 라이브러리 사용에 필요한 `Bean`들을 조건에 따라 생성하고, 기본 설정을 제공합니다.

#### B. 의존성 래퍼 모듈 (Wrapper/Starter Module) - 사용자 접점

이 모듈은 최종 사용자가 `pom.xml`에 추가하는 부분입니다.

  * **패키지 이름 규칙**: `[기술명]-spring-boot-starter` 형태를 따릅니다. (예: `my-custom-spring-boot-starter`)
  * **역할**: 실제 코드는 없으며, 자동 설정 모듈과 해당 라이브러리 자체(core library)에 대한 의존성만 모아놓는 역할을 합니다.

-----

### 2\. 🧩 Starter 제작 단계별 가이드

#### 단계 1: 프로젝트 설정 및 의존성 정의

먼저 자동 설정을 담당할 프로젝트를 만듭니다.

```xml
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.2.0</version>
</parent>

<dependencies>
    <dependency>
        <groupId>com.mycompany</groupId>
        <artifactId>my-core-library</artifactId>
        <version>1.0.0</version>
    </dependency>

    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-autoconfigure</artifactId>
    </dependency>

    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-configuration-processor</artifactId>
        <optional>true</optional>
    </dependency>
</dependencies>
```

#### 단계 2: 설정 속성 정의 (`@ConfigurationProperties`)

사용자가 `application.properties`나 `application.yml`에서 설정할 수 있는 속성을 정의합니다.

```java
package com.mycompany.starter.properties;

import org.springframework.boot.context.properties.ConfigurationProperties;

// application.yml/properties에서 my.custom-config.key로 접근 가능
@ConfigurationProperties(prefix = "my.custom-config") 
public class MyConfigProperties {
    private String apiKey;
    private int timeout = 5000; // 기본값 설정

    // Getter and Setter
    // ...
}
```

#### 단계 3: 자동 설정 클래스 작성 (`@Configuration`)

실제 빈(Bean)들을 생성하는 로직을 작성합니다.

```java
package com.mycompany.starter.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.boot.autoconfigure.condition.ConditionalOnClass; // 핵심!
import org.springframework.boot.context.properties.EnableConfigurationProperties;

@Configuration
@ConditionalOnClass(MyCoreClient.class) // 클래스가 Classpath에 있을 때만 실행
@EnableConfigurationProperties(MyConfigProperties.class) // 위에서 정의한 속성 클래스 활성화
public class MyAutoConfiguration {

    // 1. 코어 라이브러리 클래스를 Bean으로 생성하고, 설정 속성을 주입하여 초기화
    @Bean
    @ConditionalOnMissingBean // 해당 타입의 Bean이 없을 때만 생성
    public MyCoreClient myCoreClient(MyConfigProperties properties) {
        // properties.getApiKey() 등을 사용하여 초기화 로직 수행
        return new MyCoreClient(properties.getApiKey(), properties.getTimeout());
    }

    // 2. 이 외에도 필요한 다른 Bean들을 조건에 따라 생성할 수 있습니다.
}
```

#### 단계 4: 메타 정보 등록 (핵심\!)

스프링 부트에게 이 프로젝트가 자동 설정 모듈임을 알려주어야 합니다.

  * `src/main/resources/META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports` 파일을 생성합니다. (JUnit 3.0 이상)
  * 파일 내용에 자동 설정 클래스의 **풀 패키지 경로**를 한 줄로 작성합니다.

<!-- end list -->

```text
# src/main/resources/META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports
com.mycompany.starter.config.MyAutoConfiguration
```

#### 단계 5: Starter 래퍼 모듈 생성 (사용자용)

최종 사용자가 의존성으로 추가할 `my-custom-spring-boot-starter` 프로젝트를 만들고, `pom.xml`에 자동 설정 모듈(`my-spring-boot-autoconfigure`)만 의존하도록 설정합니다.

```xml
<dependencies>
    <dependency>
        <groupId>com.mycompany</groupId>
        <artifactId>my-spring-boot-autoconfigure</artifactId>
        <version>1.0.0</version>
    </dependency>
</dependencies>
```

이제 사용자는 `my-custom-spring-boot-starter` 하나만 추가하면, 스프링 부트가 자동으로 `MyAutoConfiguration`을 인식하고 필요한 `Bean`들을 생성하게 됩니다.
