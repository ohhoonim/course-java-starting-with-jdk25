# 스프링 프레임워크와 부트

## 스프링 프레임워크

스프링 프레임워크(Spring Framework)는 Java 엔터프라이즈 애플리케이션 개발을 위한 가장 널리 사용되는 오픈소스 애플리케이션 프레임워크입니다. 단순히 하나의 도구가 아니라, 애플리케이션의 전반적인 구조를 결정하는 다양한 모듈과 기능을 제공하는 <u>거대한 생태계의 기반</u>입니다.

## 스프링의 핵심 철학과 기술

- 제어의 역전 Inversion of Control, IoC
- 의존성 주입 Dependency Injection, DI

> 💡 IoC와 DI가 기억나지 않으시면 '필수 디자인 패턴'을 다시 살펴봐주세요

## 스프링 주요 모델 및 핵심 기능

 | 모듈 그룹               | 핵심 기능                   | 설명                                                                                                                                           |
 | ----------------------- | --------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
 | Core Container          | IoC, DI, Beans              | 스프링 프레임워크의 가장 근간이 되는 모듈. 객체를 생성하고 관리하는 스프링 컨테이너 기능을 제공합니다.                                         |
 | Spring MVC              | Web, REST API               | 웹 애플리케이션 및 RESTful API 구축을 위한 Model-View-Controller(MVC) 패턴을 구현합니다.                                                       |
 | Data Access/Integration | JDBC, ORM, JPA              | 데이터베이스 접근을 위한 JDBC 추상화 계층, 트랜잭션 관리, Hibernate/JPA 연동 기능을 제공합니다.                                                |
 | AOP                     | Aspect-Oriented Programming | 관점 지향 프로그래밍. 로깅, 보안, 트랜잭션 등 여러 곳에 공통적으로 적용되는 부가 기능을 핵심 비즈니스 로직과 분리하여 관리할 수 있게 해줍니다. |
 | Testing                 | JUnit, Mockito 통합         | 유닛 및 통합 테스트를 지원하며, 테스트 환경 설정을 단순화하고 Mocking 프레임워크와의 연동을 용이하게 합니다.                                   |

## 스프링 부트

스프링 프레임워크가 Java 개발의 기반 기술이라면, 스프링 부트(Spring Boot)는 이 스프링 프레임워크를 훨씬 쉽고 빠르게 시작할 수 있도록 도와주는 도구입니다.

- 자동 설정 (Auto-configuration)
  : <u>일반적인 개발 상황에 필요한 수많은 설정을 스프링 부트가 자동으로 처리해 줍니다.</u>
- 독립 실행 가능
  : Tomcat과 같은 <u>웹 서버가 내장</u>되어 있어 별도의 서버 설치 없이 Jar 파일 하나로 애플리케이션을 실행할 수 있습니다.
