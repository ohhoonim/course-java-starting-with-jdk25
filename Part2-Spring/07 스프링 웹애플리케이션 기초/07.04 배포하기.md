# 배포하기

## application.propretis

application.properties 파일은 Spring Boot 애플리케이션의 설정(Configuration) 정보를 담는 핵심 파일입니다. 이 파일은 애플리케이션이 실행될 때 필요한 환경 설정 값들을 키(Key)-값(Value) 쌍 형태로 정의하는 데 사용됩니다.

Spring Boot는 이 파일의 값을 읽어와 애플리케이션 컨텍스트를 구성하고, 개발자가 코드를 수정하지 않고도 환경별 설정을 쉽게 변경할 수 있도록 합니다.

## 서버포트 설정 

```properties
server.port=8080
```

## application.yml로 표현하기

application.properties 는 application.yml 파일 형식으로 전환하여 작성할 수 있습니다. 

| 특징        | application.properties                           | application.yml (YAML)              |
| ----------- | ------------------------------------------------ | ----------------------------------- |
| 형식        | 키=값                                            | 들여쓰기 기반의 계층 구조           |
| 가독성      | "단순하지만, 복잡한 구조에서 중복된 접두사 발생" | 계층 구조 덕분에 가독성이 높음      |
| 배열/리스트 | 인덱스나 쉼표 사용                               | - (하이픈)을 사용하여 명확하게 표현 |

- 예시 비교

| properties                 | YAML                                                                    |
| -------------------------- | ----------------------------------------------------------------------- |
| server.port=8080           | server:<br>&nbsp;&nbsp;  port: 8080                                     |
| spring.profiles.active=dev | spring:<br>&nbsp;&nbsp;profiles:<br>&nbsp;&nbsp;&nbsp;&nbsp;active: dev |


## 프로젝트 빌드하고 실행하기 : maven

```sh
# 1. 이전 빌드 결과물을 삭제하고 테스트를 건너뛰지 않고 패키징 및 로컬 설치
$ ./mvnw clean install 

# 2. 실행 가능한 JAR 파일을 생성하고 'target' 디렉터리에 저장
$ ./mvnw package

# 3. 테스트를 건너뛰고(skip tests) 컴파일만 수행
$ ./mvnw compile -DskipTests

# 4. 애플리케이션을 개발 모드로 실행 (Spring Boot 전용)
$ ./mvnw spring-boot:run

$ java -jar target/my-spring-app.jar  # Maven 빌드 결과
```


## 프로젝트 빌드하고 실행하기 : gradle 

```sh
# 1. 이전 빌드 결과물을 삭제하고 테스트를 포함하여 빌드 수행
$ ./gradlew clean build

# 2. 빌드 후 애플리케이션을 실행 (Spring Boot 전용)
$ ./gradlew bootRun

# 3. 모든 테스트를 건너뛰고(skip tests) 빌드만 수행
$ ./gradlew build -x test

# 4. 프로젝트의 모든 종속성(Dependencies)을 확인
$ ./gradlew dependencies

$ java -jar build/libs/my-spring-app.jar # Gradle 빌드 결과
```
