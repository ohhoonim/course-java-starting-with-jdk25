> spring boot 로 학생등록, 전체학생조회 api를 jdbcclient를 이용해서 코드로 작성해줘


이번에는 JPA 대신 Spring Boot 3.2부터 새롭게 도입된 `JdbcClient`를 사용하여 동일한 기능을 구현한 코드입니다.

`JdbcClient`는 기존의 `JdbcTemplate`보다 유연하고 유려한(Fluent) API 스타일을 제공하여, SQL 쿼리를 직접 작성하면서도 가독성 높은 코드를 짤 수 있게 해줍니다.

---

## 1. DTO (Data Transfer Object) 클래스 설계

`JdbcClient`를 사용할 때는 데이터베이스 테이블의 행(Row)과 매핑할 가벼운 자바 객체(DTO)를 사용하는 것이 좋습니다. 자바 14부터 지원하는 `record`를 사용하면 깔끔하게 작성할 수 있습니다.

```java
package com.example.demo.dto;

public record StudentDto(Integer studentId, String name, Integer age) {
}

```

---

## 2. Repository 클래스

### 🗄️ StudentJdbcRepository.java

`JdbcClient`를 주입받아 직접 SQL을 실행하는 계층입니다. `param()` 메서드를 통해 SQL 파라미터를 바인딩하고, `query().list()` 등을 통해 결과를 자바 객체 리스트로 쉽게 변환할 수 있습니다.

```java
package com.example.demo.repository;

import com.example.demo.dto.StudentDto;
import org.springframework.jdbc.core.simple.JdbcClient;
import org.springframework.stereotype.Repository;

import java.util.List;

@Repository
public class StudentJdbcRepository {

    private final JdbcClient jdbcClient;

    // 생성자 주입
    public StudentJdbcRepository(JdbcClient jdbcClient) {
        this.jdbcClient = jdbcClient;
    }

    /**
     * 학생 등록 (Insert)
     */
    public int save(StudentDto student) {
        String sql = "INSERT INTO student (student_id, name, age) VALUES (:studentId, :name, :age)";
        
        return jdbcClient.sql(sql)
                .param("studentId", student.studentId())
                .param("name", student.name())
                .param("age", student.age())
                .update(); // 영향받은 Row 수를 반환 (성공 시 1)
    }

    /**
     * 전체 학생 조회 (Select All)
     */
    public List<StudentDto> findAll() {
        String sql = "SELECT student_id, name, age FROM student";
        
        return jdbcClient.sql(sql)
                .query((rs, rowNum) -> new StudentDto(
                        rs.getInt("student_id"),
                        rs.getString("name"),
                        rs.getInt("age")
                ))
                .list();
    }
}

```

## 3. Service 클래스

### 🛠️ StudentService.java

컨트롤러와 리포지토리 사이에서 비즈니스 로직을 연결합니다.

```java
package com.example.demo.service;

import com.example.demo.dto.StudentDto;
import com.example.demo.repository.StudentJdbcRepository;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.List;

@Service
public class StudentService {

    private final StudentJdbcRepository studentRepository;

    public StudentService(StudentJdbcRepository studentRepository) {
        this.studentRepository = studentRepository;
    }

    @Transactional
    public void registerStudent(StudentDto studentDto) {
        int result = studentRepository.save(studentDto);
        if (result != 1) {
            throw new RuntimeException("학생 등록에 실패했습니다.");
        }
    }

    public List<StudentDto> getAllStudents() {
        return studentRepository.findAll();
    }
}

```

## 4. Controller 클래스

### 🕹️ StudentController.java

클라이언트의 HTTP 요청을 처리하는 엔드포인트입니다.

```java
package com.example.demo.controller;

import com.example.demo.dto.StudentDto;
import com.example.demo.service.StudentService;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.util.List;
import java.util.Map;

@RestController
@RequestMapping("/api/v1/students")
public class StudentController {

    private final StudentService studentService;

    public StudentController(StudentService studentService) {
        this.studentService = studentService;
    }

    /**
     * 1. 학생 등록 API
     * POST http://localhost:8080/api/v1/students
     */
    @PostMapping
    public ResponseEntity<Map<String, Object>> createStudent(@RequestBody StudentDto studentDto) {
        studentService.registerStudent(studentDto);
        
        Map<String, Object> response = Map.of(
            "message", "학생 정보가 성공적으로 등록되었습니다.",
            "student_id", studentDto.studentId()
        );
        
        return ResponseEntity.status(HttpStatus.CREATED).body(response);
    }

    /**
     * 2. 전체 학생 조회 API
     * GET http://localhost:8080/api/v1/students
     */
    @GetMapping
    public ResponseEntity<List<StudentDto>> getAllStudents() {
        List<StudentDto> students = studentService.getAllStudents();
        return ResponseEntity.ok(students);
    }
}

```

> **💡 JPA 코드와의 차이점은 무엇인가요?**
> * **JPA:** `@Entity` 매핑 정보를 바탕으로 SQL을 자바가 자동으로 생성해 줍니다. 복잡한 연관관계 처리에 유리합니다.
> * **JdbcClient (JDBC):** SQL을 직접 다루기 때문에 데이터베이스 성능 최적화나 복잡한 통계성 쿼리를 짤 때 직관적이고 유리합니다. Spring Boot 3.2 이상이라면 기존 `JdbcTemplate`보다 훨씬 간결하게 쓸 수 있어 추천하는 방식입니다.
