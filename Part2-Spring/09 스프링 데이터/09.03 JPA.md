# JPA


JPA (Java Persistence API)는 Java 애플리케이션에서 객체(Object)와 관계형 데이터베이스(RDB)의 데이터를 매핑(Mapping)하고 관리하는 표준 기술입니다. 이는 개발자가 SQL 코드를 직접 작성하지 않고도 객체 지향적인 방식으로 데이터를 다룰 수 있도록 돕는 자바의 표준 명세(Specification)입니다.

가장 중요한 역할은 ORM (Object-Relational Mapping) 기술을 사용하여 객체와 데이터베이스 테이블 사이의 패러다임 불일치(Impedance Mismatch)를 해결하는 것입니다.

## JPA의 핵심 역할: ORM

  * 객체 지향 프로그래밍 vs. 관계형 데이터베이스: Java는 객체 지향적인데 반해, RDB는 데이터를 테이블 구조로 저장합니다. JPA는 이 둘 사이의 변환을 담당합니다.

      * 개발자는 객체를 마치 자바 컬렉션처럼 다룹니다.
      * JPA 구현체(예: Hibernate)가 이 객체를 SQL로 변환하여 데이터베이스에 저장하거나 가져옵니다.

  * 반복적인 SQL 및 JDBC 코드 제거: 데이터를 조회, 삽입, 수정, 삭제(CRUD)하는 반복적인 JDBC/SQL 작업을 JPA가 대신 처리하여 개발 생산성을 높여줍니다.

## JPA와 Hibernate의 관계

JPA는 인터페이스나 명세일 뿐이며, 실제로 동작하기 위해서는 그 명세를 구현한 구현체가 필요합니다.

| 분류                   | 이름                   | 역할                                                                         |
| :--------------------- | :--------------------- | :--------------------------------------------------------------------------- |
| 명세(Specification)    | JPA                    | Java 표준이며, ORM을 위한 표준 API와 규칙을 정의합니다.                      |
| 구현체(Implementation) | Hibernate, EclipseLink | JPA 명세에 따라 실제로 객체와 DB를 매핑하고 SQL을 생성하는 라이브러리입니다. |

Spring Data JPA를 사용하면, JPA 명세를 이용하는 동시에 내부적으로 Hibernate 같은 구현체를 사용하여 데이터베이스 작업을 처리합니다.

## JPA 예제: 사용자 정보 테이블 관리 

Spring Boot 환경에서 JPA를 사용하여 `User` 정보를 데이터베이스에 저장하고 조회하는 예시입니다.

### 준비: 의존성 추가

Spring Boot 프로젝트에서 JPA를 사용하려면 다음 의존성이 필요합니다.

```groovy
// build.gradle (예시)
dependencies {
    // Spring Data JPA (JPA 명세 + Hibernate 구현체 포함)
    implementation 'org.springframework.boot:spring-boot-starter-data-jpa'
    // H2 Database (개발 및 테스트용 인메모리 DB)
    runtimeOnly 'com.h2database:h2' 
}
```

### 엔티티 클래스 (User Entity)

데이터베이스의 테이블과 1:1로 매핑되는 자바 객체입니다.

```java
import jakarta.persistence.Entity;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.GenerationType;
import jakarta.persistence.Id;

@Entity // 1. 이 클래스가 JPA 엔티티이며 DB 테이블과 매핑됨을 명시
public class User {

    @Id // 2. 테이블의 기본 키(Primary Key)를 지정
    @GeneratedValue(strategy = GenerationType.IDENTITY) // 기본 키 자동 생성 전략 (MySQL의 AUTO_INCREMENT와 유사)
    private Long id;

    private String name;
    
    private String email;

    // --- 생성자 (JPA 사용 시 기본 생성자는 필수) ---
    protected User() {}

    public User(String name, String email) {
        this.name = name;
        this.email = email;
    }

    // --- Getter 및 Setter (생략 가능, Lombok 사용 권장) ---
    public Long getId() { return id; }
    public String getName() { return name; }
    public String getEmail() { return email; }
    // ...
}
```

### Repository 인터페이스

JPA의 기능을 사용하여 엔티티를 관리(CRUD)하는 인터페이스입니다. Spring Data JPA가 이 인터페이스를 보고 실행 시점에 구현체를 자동으로 생성해줍니다.

```java
import org.springframework.data.jpa.repository.JpaRepository;
import java.util.Optional;

// JpaRepository를 상속받아 기본적인 CRUD 메소드를 자동으로 제공받음
// <엔티티 타입, ID 타입>
public interface UserRepository extends JpaRepository<User, Long> {

    // 쿼리 메소드: JPA 규칙에 따라 메소드 이름만으로 쿼리를 자동 생성
    Optional<User> findByEmail(String email);
}
```

### 사용 예시 (Service Layer)

Service 클래스에서 `UserRepository`를 사용하여 데이터베이스 작업을 수행합니다. SQL 코드는 전혀 작성되지 않습니다.

```java
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

@Service
public class UserService {

    private final UserRepository userRepository;

    @Autowired
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    // 사용자 저장 (INSERT)
    public User saveUser(User user) {
        // save() 메소드를 호출하면 JPA가 INSERT SQL을 자동 생성하여 실행
        return userRepository.save(user); 
    }

    // ID로 사용자 조회 (SELECT)
    public Optional<User> findUserById(Long id) {
        // findById() 메소드를 호출하면 JPA가 SELECT SQL을 자동 생성하여 실행
        return userRepository.findById(id); 
    }
    
    // 이메일로 사용자 조회 (Custom 쿼리 메소드)
    public Optional<User> findUserByEmail(String email) {
        // findByEmail() 메소드를 호출하면 JPA가 "SELECT * FROM User WHERE email = ?" SQL을 자동 생성
        return userRepository.findByEmail(email); 
    }
}
```
