# url로 접근 제어하기

## 실습 


### 실습 1) Spring Security URL 접근 제어  

- 다음의 시나리오를 만족하는 접근 제어 설정코드를 작성하시오.
- 역할기반접근제어(RBAC)에 대해 조사하시오.
 
---
### 1\. ⚙️ 시나리오 정의

| URL 패턴         | 필요 권한              | 설명                                                        |
| :--------------- | :--------------------- | :---------------------------------------------------------- |
| **`/public/**`** | **`permitAll()`**      | 인증 없이 누구나 접근 가능 (예: 로그인 페이지, 정적 파일)   |
| **`/user/**`**   | **`hasRole('USER')`**  | `USER` 권한을 가진 사용자만 접근 가능                       |
| **`/admin/**`**  | **`hasRole('ADMIN')`** | `ADMIN` 권한을 가진 사용자만 접근 가능                      |
| **그 외**        | **`authenticated()`**  | `USER`나 `ADMIN` 권한이 없더라도, **인증만 되면** 접근 가능 |

-----

### 2\. 💻 Spring Security 설정 코드

`@Configuration` 클래스에 `SecurityFilterChain` 빈을 정의하여 접근 규칙을 설정합니다.

```java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;
import org.springframework.security.crypto.password.PasswordEncoder;
import org.springframework.security.web.SecurityFilterChain;

@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable()) // API 서버에서는 보통 CSRF 비활성화
            
            // 1. URL 접근 권한 설정 시작
            .authorizeHttpRequests(authorize -> authorize
                // 1.1. 누구나 접근 가능 (인증 불필요)
                .requestMatchers("/public/**", "/login", "/").permitAll()

                // 1.2. 특정 역할(ROLE)만 접근 가능
                // Spring Security에서 hasRole('ADMIN')을 사용하면, 내부적으로 ROLE_ADMIN을 찾습니다.
                .requestMatchers("/admin/**").hasRole("ADMIN") 
                .requestMatchers("/user/**").hasRole("USER")

                // 1.3. 그 외 모든 요청 (위에서 정의되지 않은 모든 URL)
                .anyRequest().authenticated() // 인증된 사용자(로그인한 사용자)만 접근 허용
            )
            
            // 2. 폼 로그인 설정 (기본 로그인 페이지 사용)
            .formLogin(form -> form
                .permitAll()
            )
            
            // 3. 로그아웃 설정
            .logout(logout -> logout
                .logoutSuccessUrl("/public/home") // 로그아웃 성공 시 이동할 URL
                .permitAll()
            );

        return http.build();
    }
    
    // 비밀번호 인코더 설정
    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }

    // 2. In-Memory UserDetailsService 빈 정의
    @Bean
    public UserDetailsService userDetailsService(PasswordEncoder passwordEncoder) {
        
        // 1. 일반 사용자 (ROLE_USER 권한 부여)
        UserDetails user = User.builder()
            .username("user")
            // 비밀번호는 반드시 passwordEncoder를 사용하여 암호화해야 함
            .password(passwordEncoder.encode("userpass")) 
            .roles("USER") // 'USER' 역할을 부여
            .build();

        // 2. 관리자 사용자 (ROLE_ADMIN 권한 부여)
        UserDetails admin = User.builder()
            .username("admin")
            .password(passwordEncoder.encode("adminpass"))
            .roles("ADMIN", "USER") // 'ADMIN'과 'USER' 역할을 모두 부여
            .build();

        // InMemoryUserDetailsManager를 사용하여 사용자 정보를 메모리에 저장
        return new InMemoryUserDetailsManager(user, admin);
    }
}
```

### 3\. 🌐 테스트용 Controller 예제

위의 설정이 실제로 동작하는지 확인할 수 있는 간단한 Controller 예제입니다.

```java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class TestController {

    @GetMapping("/")
    public String home() {
        return "Welcome Home (Public access)";
    }

    @GetMapping("/public/info")
    public String publicInfo() {
        return "This is public information.";
    }

    @GetMapping("/user/dashboard")
    public String userDashboard() {
        return "User Dashboard: Access Granted for ROLE_USER.";
    }

    @GetMapping("/admin/settings")
    public String adminSettings() {
        return "Admin Settings: Access Granted for ROLE_ADMIN.";
    }
    
    @GetMapping("/data/protected")
    public String protectedData() {
        // hasRole('USER')나 hasRole('ADMIN')에 명시되지 않은 경로입니다.
        // anyRequest().authenticated() 규칙에 따라 로그인만 하면 접근 가능합니다.
        return "Protected Data: Only requires successful login.";
    }
}
```
