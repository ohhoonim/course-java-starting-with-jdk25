# config bean 등록하기

## Spring Security

Spring Security는 Spring 기반 애플리케이션의 인증(Authentication)과 인가(Authorization), 그리고 일반적인 보안 취약점 방어를 위한 프레임워크입니다. 

Spring Security는 서블릿 필터(Servlet Filter) 체인을 통해 동작하며, 들어오는 모든 HTTP 요청을 가로채서 보안 검사를 수행합니다.

- 인증 (Authentication) : '누구인지' 확인하는 과정입니다.
  - 사용자가 제공한 자격 증명(<u>Credentials</u>), 즉 사용자 이름과 비밀번호 등이 유효한지 확인합니다.
  - 성공적으로 인증되면, 해당 사용자의 주요 정보(<u>Principal</u>)를 <u>Security Context</u>에 저장합니다.

- 인가 (Authorization) : '무엇을 할 수 있는지' 확인하는 과정입니다.
  - 인증된 사용자가 특정 자원(URL, 메소드 등)에 접근할 권한(<u>Role/Authority</u>)이 있는지 검사합니다.

## Servlet stack

여기서는 Servlet Stack을 기준으로 설명합니다.

> 공식 가이드 : https://docs.spring.io/spring-security/reference/servlet/index.html
> 
> Servlet Stack (Spring MVC)
>   - CPU 집약적인 로직이 많은 경우 (복잡한 계산) 
>   - 기존 Java/Servlet 모델에 익숙한 팀 
>   - 단순한 트랜잭션 및 비즈니스 로직 
>   - 미들웨어와의 연동(예: JTA, 기존 Spring Data JDBC)이 필수적인 경우
> 
> Reactive Stack (Spring WebFlux)
>   - I/O 지연(Latency)이 길고 높은 동시성이 요구되는 마이크로서비스 
>   - 데이터 스트리밍이 필요한 경우 (웹소켓, SSE) 
>   - 논블로킹 데이터베이스(R2DBC 등)를 사용하는 경우


## SecurityFilterChain

```java
@Configuration
@EnableWebSecurity // (Spring Boot에서는 생략 가능)
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            // 1. 인가 (Authorization) 설정
            .authorizeHttpRequests(authorize -> authorize
                .requestMatchers("/public/**").permitAll() // "/public/" 경로는 모두 허용
                .requestMatchers("/admin/**").hasRole("ADMIN") // "/admin/" 경로는 ADMIN 권한 필요
                .anyRequest().authenticated() // 그 외 모든 요청은 인증 필요
            )
            // 2. 폼 로그인 설정 (기본 로그인 페이지 사용)
            .formLogin(Customizer.withDefaults())
            // 3. CSRF 방어 비활성화 (API 서버의 경우)
            .csrf(csrf -> csrf.disable());

        return http.build();
    }

    // PasswordEncoder 빈 등록 (BCrypt 사용)
    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```
