## `@Configuration` 클래스 내 `@Bean` 중복 정의 시 동작 

하나의 `@Configuration` 클래스 내에서 동일한 이름 또는 동일한 타입의 `@Bean` 메소드를 중복 정의했을때  

### 1\. 이름이 중복된 `@Bean` 정의 (가장 흔한 경우)

Spring 컨테이너는 빈을 등록할 때 빈의 이름(Name)을 기준으로 관리합니다.

#### A. 동작 방식

  * 빈 이름 충돌: `@Configuration` 클래스 내에서 두 개 이상의 `@Bean` 메소드가 동일한 이름을 반환하거나, 명시적으로 이름을 지정하지 않아 메소드 이름이 같을 경우 충돌이 발생합니다.
  * 해결: Spring은 기본적으로 마지막에 정의된 빈을 컨테이너에 등록하고, 그 이전의 동일 이름 빈은 무시됩니다. (단, 이는 클래스 내부의 메소드 순서에 따라 컴파일러가 바이트코드를 생성하는 방식에 의존하므로 예측이 어려울 수 있습니다.)
  * 권장: 명시적으로 빈 이름 중복을 피해야 합니다.

#### B. 예시 (메소드 이름 충돌)

```java
@Configuration
public class AppConfig {
    
    // 이 메소드의 반환 객체는 'userService'라는 이름의 빈으로 등록됨
    @Bean
    public UserService userService() {
        System.out.println("First UserService created");
        return new UserService(1);
    }

    // 이 메소드도 'userService'라는 이름의 빈으로 등록되므로 충돌 발생
    @Bean
    public UserService userService() { 
        System.out.println("Second UserService created");
        return new UserService(2);
    }
}
// 결과: 컴파일 에러 발생 (자바 메소드 중복 정의)
```

> 참고: 자바 컴파일러는 하나의 클래스 내에서 메소드 시그니처(이름과 매개변수)가 완전히 동일한 메소드를 허용하지 않으므로, 사실상 컴파일 에러가 발생하여 실행 자체가 불가능합니다.

#### C. 예시 (이름 명시 후 충돌)

```java
@Configuration
public class AppConfig {
    
    // 빈 이름: 'myBean'
    @Bean(name = "myBean")
    public String createBeanA() {
        return "Value A";
    }

    // 빈 이름: 'myBean' (이름 중복)
    @Bean(name = "myBean")
    public String createBeanB() {
        return "Value B";
    }
}
// 결과: 컴파일 에러 (메소드 이름이 다르므로 컴파일 가능)
// Spring 동작: 두 번째 @Bean인 createBeanB()가 컨테이너에 'myBean'으로 등록됩니다.
// 첫 번째 빈은 무시되고 두 번째 빈이 덮어씁니다.
```

### 2\. 타입은 같지만 이름이 다른 `@Bean` 정의

이 경우, Spring은 두 빈을 모두 컨테이너에 등록합니다.

#### A. 동작 방식

  * 성공적인 등록: 두 빈은 서로 다른 이름(`beanA`, `beanB`)으로 등록되므로 충돌이 발생하지 않습니다.
  * 의존성 주입 시 문제: 나중에 `@Autowired`를 통해 이 타입(`MyInterface`)을 주입받으려 할 때, 컨테이너에 동일 타입의 빈이 두 개 이상 존재하므로 `NoUniqueBeanDefinitionException`이 발생합니다.

#### B. 해결책 (의존성 주입 시)

동일 타입 빈이 여러 개 있을 때는 주입 시점을 명확히 해야 합니다.

1.  `@Qualifier` 사용: 주입받을 빈의 이름을 명시합니다.
    ```java
    @Autowired
    @Qualifier("beanA") // 'beanA'라는 이름의 빈을 주입
    private MyInterface myInterface; 
    ```
2.  이름 기반 주입: 필드 이름을 빈 이름과 동일하게 지정합니다. (Spring의 기본 동작)
    ```java
    @Autowired
    private MyInterface beanB; // 필드 이름이 빈 이름('beanB')과 같으므로 해당 빈 주입
    ```

### 3\. 결론

`@Configuration` 클래스 내에서 `@Bean` 메소드를 정의할 때는 빈 이름이 중복되지 않도록 주의해야 합니다. 중복을 피할 수 없는 경우에는 `@Primary` 어노테이션을 사용하여 기본적으로 주입될 빈을 지정하거나, `@Qualifier`를 사용하여 명확하게 주입 대상을 지정해야 합니다.
