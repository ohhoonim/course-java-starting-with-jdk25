# 04.02 Proxy 패턴

Proxy 패턴은 실제 객체(Real Subject)에 대한 대리인(Proxy) 객체를 두어, 실제 객체로의 접근을 제어하거나 기능을 추가하는 것입니다. 클라이언트는 프록시를 통해 실제 객체를 사용하는 것처럼 보이지만, 모든 요청은 프록시를 거쳐 실제 객체로 전달됩니다.

- 접근 제어 (Access Control): 실제 객체로의 접근 권한을 확인하거나, 특정 조건에서만 접근을 허용합니다.
- 기능 추가 (Enhancement): 실제 객체가 작업을 수행하기 전후에 로깅, 캐싱, 보안, 지연 로딩 등의 부가적인 기능을 추가합니다 (Aspect-Oriented Programming, AOP의 기본 원리).
- 자원 관리: 비용이 많이 드는 객체의 생성을 지연시키거나(Lazy Initialization), 원격 객체에 대한 통신을 담당합니다.

## Proxy pattern 유형

- 가상 프록시 (Virtual Proxy): 비용이 많이 드는 객체(예: 큰 이미지 파일)의 생성을 실제로 필요할 때까지 지연시킵니다 (지연 로딩).
- 보호 프록시 (Protection Proxy): 접근 권한을 체크하여 인가된 사용자만 실제 객체의 메소드를 호출할 수 있도록 접근을 제어합니다.
- 원격 프록시 (Remote Proxy): 원격 시스템(다른 서버)에 존재하는 객체에 대한 접근을 대리합니다 (RMI, Web Service 등). 클라이언트에게는 마치 로컬 객체처럼 보이게 합니다.
- 캐싱 프록시 (Caching Proxy): 실제 객체의 메소드 실행 결과를 캐싱하여, 동일한 요청이 들어오면 실제 객체 호출 없이 저장된 결과를 바로 반환합니다.


## 보호 프록시 예제

```java

// 1. Subject 인터페이스: 실제 객체와 프록시가 공유하는 규약
interface UserManagement {
    String getUserInfo(String userId);

    void deleteUser(String userId); // 권한이 필요한 메소드
}

// 2. RealSubject: 실제 비즈니스 로직 구현
class RealUserService implements UserManagement {
    @Override
    public String getUserInfo(String userId) {
        return "사용자 ID: " + userId + ", 이름: Smith";
    }

    @Override
    public void deleteUser(String userId) {
        IO.println("=> [실제 서비스] 사용자 " + userId + "를 데이터베이스에서 삭제했습니다.");
    }
}

// 3. Proxy: 실제 객체를 보호하고 기능을 추가
class AdminUserProxy implements UserManagement {
    private final RealUserService realService;
    private final boolean isAdmin; // 프록시가 관리하는 부가 기능 (권한 상태)

    AdminUserProxy(boolean isAdmin) {
        this.realService = new RealUserService(); // 실제 객체 생성 및 포함
        this.isAdmin = isAdmin;
    }

    @Override
    public String getUserInfo(String userId) {
        // 부가 작업 1: 로깅 및 전달
        IO.println("[프록시] 사용자 정보 조회 요청 수신.");
        return realService.getUserInfo(userId);
    }

    @Override
    public void deleteUser(String userId) {
        // 부가 작업 2: 접근 제어 (핵심 로직)
        if (isAdmin) {
            IO.println("[프록시] 권한 확인 완료. 삭제 요청을 실제 서비스로 전달.");
            realService.deleteUser(userId);
        } else {
            IO.println("[프록시] 🚨 오류: 관리자 권한이 없어 삭제 작업을 거부합니다.");
        }
    }
}

void main() {
    IO.println("--- Proxy 패턴 활용 예제 ---");

    // 1. 관리자 권한을 가진 프록시 생성
    UserManagement adminClient = new AdminUserProxy(true);
    IO.println("\n[관리자 클라이언트]");
    adminClient.deleteUser("user_001"); // 삭제 성공

    IO.println("-------------------------");

    // 2. 일반 사용자 권한을 가진 프록시 생성
    UserManagement normalClient = new AdminUserProxy(false);
    IO.println("\n[일반 사용자 클라이언트]");
    normalClient.deleteUser("user_002"); // 삭제 거부

    // 조회 기능은 권한 없이도 가능
    IO.println("\n조회 결과: " + normalClient.getUserInfo("user_002"));
}

```
