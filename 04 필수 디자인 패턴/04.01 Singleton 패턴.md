# 04.01 Singleton 패턴

특정 클래스의 인스턴스가 프로그램 전체에서 오직 하나만 존재하도록 보장하고, 이 인스턴스에 대한 전역적인 접근점을 제공하는 패턴입니다.

- 단일 인스턴스 보장
- 전역 접근 

## 주요 사용 사례

- Configuration : 애플리케이션의 모든 설정 정보는 하나의 객체로 관리되어야 합니다.
- Logger : 모든 로그 메시지는 중앙 집중화된 하나의 로거 인스턴스를 통해 기록되어야 합니다.
- Connection Pool : 시스템 자원을 절약하기 위해 연결 풀은 단 하나의 인스턴스로 관리되는 것이 효율적입니다.

## 구현 예제

```java
void main() {
    IO.println("--- Singleton 패턴 활용 예제 ---");

    // 1. 유일한 인스턴스를 얻습니다.
    SettingsManager manager1 = SettingsManager.getInstance();

    // 2. 다른 곳에서 다시 인스턴스를 얻습니다.
    SettingsManager manager2 = SettingsManager.getInstance();

    // 3. 두 변수가 동일한 인스턴스를 참조하는지 확인 (== 연산자는 참조 비교)
    IO.println("manager1과 manager2는 같은 객체인가? " + (manager1 == manager2)); // true 출력

    // 상태 변경 테스트
    manager1.setAppVersion("1.0.0");
    IO.println("Manager 2가 보는 버전: " + manager2.getAppVersion()); // "1.0.0" 출력
}


static class SettingsManager {

    private String appVersion = "0.0.0";

    // 1. private 생성자: 외부에서 new를 통한 인스턴스 생성을 막습니다.
    private SettingsManager() {
        IO.println("SettingsManager 인스턴스가 생성되었습니다.");
    }

    // 2. 정적 내부 클래스(Holder)를 통한 인스턴스 생성
    // - Holder 클래스는 getInstance()가 호출될 때까지 로드되지 않습니다 (지연 초기화).
    // - JVM의 클래스 로딩 메커니즘을 이용하므로 스레드 안전성이 보장됩니다.
    private static class SingletonHolder {
        private static final SettingsManager INSTANCE = new SettingsManager();
    }

    // 3. public static 접근 메소드: 유일한 인스턴스를 반환합니다.
    public static SettingsManager getInstance() {
        return SingletonHolder.INSTANCE;
    }

    // 비즈니스 로직
    public String getAppVersion() {
        return appVersion;
    }

    public void setAppVersion(String version) {
        this.appVersion = version;
    }
}
```
